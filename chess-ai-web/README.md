# 접근 가능한 MCTS 체스

브라우저에서 바로 실행되는 정적 호스팅형 체스 웹앱입니다. 기본 엔진은 **PUCT 기반 MCTS**를 중심으로 하고, 여기에 **경량 정적 평가기**, **전술 확인 탐색**, **소형 오프닝 북**, **Transposition Table**을 결합한 하이브리드 구조를 사용합니다.

이 프로젝트는 다음 목표를 우선시했습니다.

- 정적 호스팅 가능성(GitHub Pages 포함)
- 키보드만으로 완전 조작 가능한 접근성
- `Stockfish`류의 깊이 우선 계열과 다른, **유망한 수 우선 확장** 중심의 엔진 구조
- 브라우저 메모리 한도 안에서 유지 가능한 번들 크기
- 나중에 더 강한 평가기(예: TF.js/WASM 기반 소형 신경망)로 교체하기 쉬운 구조

## 주요 기능

- **표(`<table>`) 기반 체스판**
  - 64칸 전체가 순차 탐색 가능한 버튼입니다.
  - 각 칸의 접근 가능한 이름은 `기물 색상 + 기물 이름 + 좌표` 순서로 제공됩니다.
  - 예: `백 킹, e1`, `흑 룩, a8`, `빈칸, e4`
- **키보드 전용 조작 지원**
  - `Tab`으로 모든 칸과 컨트롤에 접근 가능
  - `Enter` / `Space`로 선택 및 이동
  - 보드 안에서는 방향키로 인접 칸 포커스 이동 가능
- **선택/이동/잡기/체크/종료 상태 라이브 안내**
  - `aria-live` 영역으로 자동 공지
  - 선택 상태는 `aria-pressed`로 노출
- **난이도 프리셋 + 사용자 지정**
  - `입문 / 보통 / 강함 / 매우 강함 / 사용자 지정`
  - 직접 파라미터 입력은 **사용자 지정**일 때만 활성화
- **브라우저 워커 기반 AI 탐색**
  - UI 스레드와 탐색 스레드를 분리
  - 탐색 취소 시 Worker를 재생성해 대기열 꼬임 방지

## 실행 방법

### 1. 웹앱만 실행

이 앱은 빌드 없이 바로 정적 호스팅할 수 있습니다.

```bash
python3 -m http.server 4173
```

그 뒤 브라우저에서 `http://localhost:4173`로 접속하십시오.

> `file://`로 직접 열면 브라우저의 모듈 Worker 보안 정책 때문에 일부 환경에서 동작하지 않을 수 있습니다. 반드시 HTTP 서버 또는 GitHub Pages 같은 정적 호스팅으로 실행하는 것을 권장합니다.

### 2. GitHub Pages 배포

가장 단순한 방법은 저장소 루트에 이 프로젝트 파일들을 그대로 두고 Pages를 활성화하는 것입니다.

1. 저장소에 파일 업로드
2. GitHub Pages를 **Deploy from a branch**로 설정
3. 브랜치 `main`(또는 원하는 브랜치) / 폴더 `/root` 선택
4. 배포 완료 후 제공된 URL 접속

`.nojekyll` 파일이 포함되어 있으므로 GitHub Pages의 기본 Jekyll 처리와 무관하게 정적 파일을 그대로 배포할 수 있습니다.

## 테스트 실행

에이전트 기반 브라우저 테스트는 Playwright Core를 사용합니다.

```bash
npm install
npm run test:agent
```

테스트 실행 시 Chromium 실행 파일을 찾지 못하면 `CHROMIUM_PATH` 환경 변수를 지정하십시오.

```bash
CHROMIUM_PATH=/usr/bin/chromium npm run test:agent
```

## 조작 방법

1. 색상과 난이도를 선택합니다.
2. `새 게임 시작`을 누릅니다.
3. 기물이 있는 칸으로 이동하여 `Enter` 또는 `Space`를 누르면 선택됩니다.
4. 목적지 칸에서 다시 `Enter` 또는 `Space`를 누르면 이동됩니다.
5. 프로모션이 필요하면 승진 기물 버튼이 나타납니다.
6. `한 턴 되돌리기`와 `AI 착수`로 보조 조작이 가능합니다.

## 난이도 구성

기본 난이도는 시간과 시뮬레이션 수를 함께 바꾸는 프리셋 방식입니다. 단순히 시간만 늘리는 대신, 루트 온도와 PUCT 계수, progressive widening 파라미터도 함께 조정해 플레이 스타일 차이를 만들었습니다.

- **입문**: 짧은 탐색 시간, 높은 온도, 낮은 전술 깊이
- **보통**: 균형형
- **강함**: 더 긴 탐색, 더 좁은 온도, 전술 확인 강화
- **매우 강함**: 가장 긴 탐색과 더 공격적인 후보 정제
- **사용자 지정**: 세부 파라미터 직접 조정

## 엔진 개요

이 엔진은 `Stockfish`류의 깊이 우선 알파-베타 검색을 메인으로 삼지 않습니다. 우선 구조는 다음과 같습니다.

1. **소형 오프닝 북** 확인
2. 루트 노드 확장
3. **PUCT**로 유망 자식 선택
4. **progressive widening**으로 한 번에 모든 수를 다 열지 않고 점진 확장
5. 리프 평가 시 **정적 평가 + 얕은 전술 negamax/quiescence** 혼합
6. 루트에서는 방문 수와 온도에 따라 수 선택

현재 구현은 브라우저 환경 제약 때문에 대형 신경망 가중치를 직접 탑재하지 않았습니다. 대신 다음을 통해 기력을 끌어올렸습니다.

- 이동 prior 휴리스틱
- 중앙 장악/기물 전개/폰 구조/룩 파일/킹 안전 기반 정적 평가
- 캡처/승진/캐슬링/체크 중심 전술 탐색
- 소형 오프닝 레퍼토리
- Transposition Table

이 구조 덕분에 엔진의 **첫 번째 기준은 최선 우선 확장**이고, 전통적인 깊이 우선 탐색은 **보조 전술 확인 레이어**로만 쓰입니다.

## 접근성 요구사항 대응 요약

- 모든 상호작용 항목에 접근 가능한 이름 제공
- 64칸 전체 Tab 탐색 가능
- 기물 칸은 버튼으로 동작하고 빈칸도 동일한 조작 방식 제공
- 선택 상태는 `aria-pressed`와 접근 가능한 이름의 `선택됨`으로 제공
- 이동/잡기/체크/게임 종료는 `aria-live`로 공지
- 좌표 입력 없이 순차 탐색만으로 판 조회 및 이동 가능

## 확장 포인트

- `StaticEvaluator`를 TF.js/WASM 기반 `NeuralEvaluator`로 교체
- `MovePrior`를 정책망 출력으로 대체
- 오프닝 북 확장
- 엔드게임 테이블베이스 연동(브라우저 메모리 한도 주의)
- 더 정교한 시간 관리 및 반복심화형 보조 탐색 추가
