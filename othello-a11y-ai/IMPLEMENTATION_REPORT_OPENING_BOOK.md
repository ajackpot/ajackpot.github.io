# 오프닝북 통합 보고서

## 요약
- 이름 있는 오델로 오프닝 카탈로그를 바탕으로 소형 오프닝북을 추가했습니다.
- 대칭 중복을 제거한 seed line은 99개이며, 런타임 전개 후 실제 조회 가능한 포지션 엔트리는 447개입니다.
- 오프닝북은 초기 12플라이까지 직접 수를 제안하고, 최대 18플라이까지는 탐색 루트 수 정렬 참고 정보로도 활용할 수 있게 설계했습니다.
- 초기 위치에서는 엔진이 탐색 없이 즉시 북 수를 반환합니다.

## 데이터 선정 방식
1. Robert Gatliff 오프닝 카탈로그의 이름 있는 라인과 출현 빈도 정보를 기준 데이터로 삼았습니다.
2. move sequence 목록 396개를 초기 국면의 색 보존 대칭(항등, 180도 회전, 주대각 반사, 부대각 반사) 기준으로 정리했습니다.
3. 그 결과 정확히 99개의 고유 seed line이 남았고, 이를 compact opening book의 입력으로 채택했습니다.
4. 각 포지션에서는 가능한 다음 수마다 누적 빈도와 대표 오프닝 이름을 함께 저장합니다.

## 구현 세부 사항
### 추가 파일
- `js/ai/opening-book-data.js`
  - 99개 seed line과 이름, 빈도 정보를 보관합니다.
- `js/ai/opening-book.js`
  - seed line을 실제 포지션 엔트리로 전개하고 조회하는 모듈입니다.
  - 런타임에 한 번만 빌드되며 캐시됩니다.

### 엔진 통합
- `js/ai/search-engine.js`
  - 초기~초중반에는 `lookupOpeningBook(state)`로 북 히트를 확인합니다.
  - 히트가 있으면 스타일/위험도/기동성까지 반영한 경량 점수로 북 후보를 정렬합니다.
  - 직접 사용 구간(<= 12 ply)에서는 탐색 없이 즉시 수를 반환합니다.
  - 그 이후 구간에서는 full search를 유지하되, 북 후보가 있으면 루트 move ordering 보너스를 줍니다.
  - 결과 객체에 `source: 'opening-book' | 'search'` 와 `bookHit` 메타데이터를 추가했습니다.

### UI 반영
- `js/ui/formatters.js`
  - 최근 AI 탐색 요약에서 오프닝북 직선택 여부와 대표 계열을 표시합니다.
- `README.md`
  - 소형 오프닝북 구조와 추가 파일을 문서화했습니다.

## 구현 중 확인한 핵심 포인트
- 초기 보드는 8개 전체 정다면체 대칭이 아니라, **색을 보존하는 4개 대칭**만 그대로 적용 가능합니다.
- 90도 회전이나 수평/수직 반사는 시작 배치에서 흑백 돌을 서로 교환해 버리므로, seed line 전개에 그대로 쓰면 불법 수가 생깁니다.
- 따라서 색을 보존하는 4개 대칭만 사용해 전개했습니다.

## 검증 결과
### 오프닝북 요약
- seed line 수: 99
- 전개 후 포지션 수: 447
- 최대 북 깊이: 18 ply

### 동작 예시
- 초기 위치에서 결과:
  - `source = opening-book`
  - 탐색 노드 `0`
  - `bookHits = 1`, `bookMoves = 1`
- 중반 테스트 위치(17 ply)에서는:
  - `source = search`
  - 깊이 5까지 정상 탐색
  - 오프닝북에 과도하게 의존하지 않고 기존 탐색기로 복귀

## 비고
- 이번 구현은 “대형 토너먼트 북”이 아니라 브라우저 정적 앱에 적합한 compact opening book입니다.
- 더 확장하려면 다음 단계에서 토너먼트 기보(예: THOR/WTHOR 계열) 기반 자동 확장이나, 오프닝북 후보의 엔진 재평가 캐시를 추가하는 것이 자연스럽습니다.
