# 24단계 보고서 · 예약 확인 흐름과 검색 저장 옵션 정리

## 1. 작업 배경

이번 단계는 예약 캘린더와 검색 결과 목록에서 실제 서비스처럼 보이는 흐름을 더 명확하게 만들고, 과업 문구와 화면 조작 사이의 불필요한 혼란을 줄이기 위해 진행했다.

주요 요청은 다음과 같았다.

- 예약 캘린더 첫 과업의 문구에서 문자 알림을 토글처럼 오해할 수 있는 표현을 고친다.
- 예약 시간 확인 화면의 `예약 확정` 버튼을 `예약하기`로 바꾸고, 실제 예약 전 최종 확인 대화상자를 한 번 더 둔다.
- 상담 옵션이 필요한 일정은 옵션을 선택한 뒤 최종 예약 확인 대화상자로 이어지게 한다.
- 검색 결과 목록 저장 과업에서 `저장 위치`와 `저장 형식`이 혼동되지 않도록 옵션을 단순화한다.
- 검색 결과 목록 비교안 B의 바로가기 링크를 글로벌 탐색 메뉴보다 앞, 수행 화면 최상단에 배치한다.

## 2. 예약 캘린더 변경

### 2.1 과업 문구 수정

기존 문구는 다음과 같이 문자 알림을 별도 수신 토글처럼 해석할 여지가 있었다.

> 문자 알림은 받는 것으로 선택하십시오.

이를 다음처럼 바꿨다.

> 김민아 상담사의 화요일 14:30 비대면 30분 시간으로 예약하고, 상담 주제는 직장 스트레스, 알림 방법은 문자 알림으로 선택하고, 사전 작성란은 비워두십시오.

변경 파일:

- `data/tasks-calendar.js`

### 2.2 예약 버튼명 정리

예약 시간 확인 대화상자에서 즉시 확정처럼 들리는 `예약 확정`, `새 예약 확정` 문구를 제거하고 `예약하기`로 통일했다.

이제 예약 시간 버튼 실행 흐름은 다음과 같다.

#### 상담 옵션이 없는 일정

1. 예약 시간 실행
2. 예약 확인 대화상자 표시
3. `예약하기` 실행
4. 최종 예약 확인 대화상자 표시
5. `예약하기` 실행
6. 예약 완료 화면 표시

#### 상담 옵션이 있는 일정

1. 예약 시간 실행
2. 예약 확인 대화상자 표시
3. `예약하기` 실행
4. 상담 옵션 선택 대화상자 표시
5. 상담 주제, 알림 방법, 사전 작성란 확인
6. `선택 완료하고 예약하기` 실행
7. 최종 예약 확인 대화상자 표시
8. `예약하기` 실행
9. 예약 완료 화면 표시

### 2.3 최종 예약 확인 대화상자 추가

새 대화상자 종류 `booking-final-confirm`을 추가했다. 이 대화상자는 선택한 일정을 다시 보여 주고 실제 예약을 진행할지 묻는다.

추가된 동작:

- `openBookingFinalConfirmModal()`
- `confirmBookingFinalFromModal()`
- `data-action="dialog-finalize-booking"`

기존 예약 완료 판정은 `finalizeBookingForSlot()`에서 계속 처리하므로, 내부 성공 판정은 기존 구조와 호환된다.

### 2.4 실패 메시지 문구 보정

최종 판정에서 기존의 `예약 확정 화면에 진입하지 못했습니다.`는 현재 화면 구조와 맞지 않아 `예약 완료 화면에 진입하지 못했습니다.`로 바꿨다.

## 3. 검색 결과 목록 변경

### 3.1 저장 옵션 단순화

기존 저장 옵션은 다음 3개 묶음이었다.

- 저장 위치
- 포함할 내용
- 저장 형식

이 구조는 `개인 보관함에 저장`과 `웹에서 보기`가 서로 다른 의미인지 혼동을 만들 수 있었다. 이번 단계에서는 `저장 위치`와 `저장 형식`을 하나의 `저장 위치` 선택으로 합쳤다.

변경 후 저장 옵션은 다음 2개 묶음이다.

#### 저장 위치

- 개인 보관함에 저장
- 공유 보관함에 저장
- PDF 파일로 내보내기
- 텍스트 파일로 내보내기

#### 포함할 내용

- 요약만 포함
- 상담 전 점검 항목 포함
- 전체 내용 포함

과업 성공 기준도 더 이상 `format: 'web'`을 요구하지 않는다. `개인 보관함에 저장`과 `상담 전 점검 항목 포함`만 맞으면 된다.

변경 파일:

- `data/tasks-search.js`
- `search-app.js`

### 3.2 비교안 B 바로가기 위치 조정

검색 결과 목록 비교안 B의 `검색 결과 영역으로 바로가기`, `검색 조건 설정으로 바로가기` 링크가 글로벌 탐색 메뉴 뒤에 나오던 문제를 수정했다.

이제 수행 화면 렌더링 순서는 다음과 같다.

1. 선택한 경우 과업 요청 사항 카드
2. 검색 결과/검색 조건 바로가기 링크
3. 검색 서비스 머리글과 글로벌 탐색 메뉴
4. 보조 패널
5. 검색 조건
6. 검색 결과

이렇게 해서 바로가기 링크가 실제로 첫 진입 비용을 줄이는 역할을 하도록 했다.

## 4. 벤치마크 갱신

예약 캘린더는 최종 예약 확인 대화상자가 추가되었으므로 예약 확인 단계의 조작 비용을 보정했다. 검색 결과 목록은 저장 옵션 수가 줄어들었으므로 저장 옵션 선택 단계의 조작 비용을 낮췄다.

갱신된 예상 합계는 다음과 같다.

### 예약 캘린더

- 키보드 사용자: A 129.1초 → B 55.1초
- 화면낭독 사용자: A 229.2초 → B 93.3초
- 스위치 사용자: A 337.1초 → B 135.2초

### 검색 결과 목록

- 키보드 사용자: A 77.2초 → B 34.8초
- 화면낭독 사용자: A 133.6초 → B 58.5초
- 스위치 사용자: A 190.1초 → B 87.0초

변경 파일:

- `data/benchmark-graphs-calendar.js`
- `data/benchmark-graphs-search.js`
- `data/benchmark-results-calendar.json`
- `data/benchmark-results-search.json`

## 5. 문서와 점검 항목 갱신

다음 문서를 현재 흐름에 맞게 갱신했다.

- `README.md`
- `docs/manual-release-checklist.md`
- `docs/language-guideline-wcag-3.1.3-kwcag.md`

추가한 점검 관점:

- 상담 옵션 일정이 옵션 선택 뒤 최종 예약 확인 대화상자로 이어지는지

- `PDF 파일로 내보내기`, `텍스트 파일로 내보내기`가 저장 위치 선택지로 제공되는지


## 7. 남은 수동 점검

브라우저와 실제 보조기술 조합에서는 다음을 우선 확인해야 한다.

- 김민아 상담사 예약에서 `예약하기 → 상담 옵션 선택 → 선택 완료하고 예약하기 → 최종 예약 확인 → 예약하기 → 예약 완료 화면` 순서가 자연스러운지
- 박하늘 상담사 예약에서 상담 옵션 없이 `예약하기 → 최종 예약 확인 → 예약하기 → 예약 완료 화면`으로 이어지는지
- 검색 결과 목록 비교안 B에서 바로가기 링크가 글로벌 탐색 메뉴보다 먼저 초점을 받는지
- 검색 저장 옵션에서 저장 위치 4개와 포함할 내용 선택이 혼동 없이 읽히는지
