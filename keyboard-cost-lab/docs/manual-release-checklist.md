# 배포 전 수동 점검표

부제: 브라우저 자동화의 한계를 보완하기 위한 서비스별 배포 점검표

## 1. 목적

이 문서는 브라우저 자동화로 끝까지 검증하기 어려운 항목을 사람이 직접 확인할 수 있도록 만든 배포용 점검표다. 홈 화면, 서비스 진입, 과업 준비, 새 탭 수행, 결과 저장, 문구 이해 가능성까지 한 번에 점검한다.

## 2. 공통 준비

- [ ] `node scripts/run-benchmark.mjs`를 다시 실행해 사전 계산 결과 파일을 최신 상태로 만들었다.
- [ ] 정적 서버에서 `index.html`, `comments.html`, `product.html`이 모두 열리는지 확인했다.
- [ ] 새 탭 열기 기능이 브라우저의 팝업 차단에 막히지 않는지 확인했다.
- [ ] `docs` 폴더에 이번 단계 보고서와 이전 단계 보고서가 함께 남아 있는지 확인했다.
- [ ] 용어 가이드 문서가 최신 상태인지 확인했다.

## 3. 공통 화면 흐름 점검

### 3.1 홈 화면

- [ ] 첫 화면에서 바로 과업 시작 버튼이 보이지 않는다.
- [ ] 서비스 유형을 먼저 고른 뒤 해당 서비스 화면으로 들어갈 수 있다.
- [ ] 현재 공개된 서비스 유형이 명확하게 보인다.
- [ ] 홈 화면 문구에 불필요한 영어가 남아 있지 않다.

### 3.2 서비스 화면

- [ ] 서비스 요약 문단만 읽어도 무엇을 비교하는 실험인지 이해할 수 있다.
- [ ] `과업 준비로 이동`, `서비스 선택으로 돌아가기` 문구가 동작을 분명하게 설명한다.
- [ ] 용어 설명 보기에서 남겨 둔 용어의 뜻을 확인할 수 있다.

### 3.3 과업 준비 화면

- [ ] 과업 시작 전에 목표, 단계, 현재 상태, 목표 대상이 모두 메인 창에 표시된다.
- [ ] 수행 탭을 열기 전에도 과업 정보를 충분히 읽을 수 있다.
- [ ] 새 탭을 열어도 원래 메인 창이 그대로 남아 있어 과업 내용을 다시 확인할 수 있다.
- [ ] 상태 메시지가 `탭 열림`, `실제 계측 시작`, `탭 닫힘`, `결과 전달 완료`를 구분해 보여 준다.

### 3.4 수행 탭 계측 규칙

- [ ] 수행 탭을 열자마자 시간 기록이 시작되지 않는다.
- [ ] 첫 키 입력이나 첫 클릭이 들어가는 순간부터 기록이 시작된다.
- [ ] 수행 탭을 다른 탭으로 가려 두었을 때 숨김 시간이 결과에서 빠진다.
- [ ] 수행 탭 하단의 첫 이동 버튼과 `이 탭 닫기` 버튼 조작은 실제 지표에 포함되지 않는다.

### 3.5 언어와 이해 가능성

- [ ] `slot`, `skip link`, `modal`, `focus` 같은 낯선 영어가 화면에 그대로 남아 있지 않다.
- [ ] 버튼과 링크 문구만 읽어도 눌렀을 때 일어나는 결과를 예상할 수 있다.
- [ ] 같은 뜻의 용어가 페이지마다 달라지지 않는다.
- [ ] 설명이 필요한 용어는 용어 설명 보기나 인접 문구에서 뜻을 확인할 수 있다.

## 4. 예약 캘린더 점검표

### 4.1 공통

- [ ] 홈 화면의 `예약 캘린더` 카드에서 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면에서 현재 예약 내용과 목표 예약 시간이 보인다.
- [ ] 수행 탭 하단에는 `결과로 바로 이동`, `이 탭 닫기` 두 버튼만 보인다.

### 4.2 비교안 A

- [ ] 상단 보조 링크와 조건 선택을 지나야 예약 가능 시간에 도달한다.
- [ ] 예약 시간마다 `선택`, `시간 안내 보기`가 따로 있어 순차 이동이 길다.
- [ ] 대화상자를 닫으면 방금 보던 예약 시간 대신 `예약 가능 시간` 제목 근처로 돌아온다.
- [ ] 과업 1, 2, 3이 모두 끝나면 결과 요약 화면으로 넘어간다.

### 4.3 비교안 B

- [ ] `결과로 바로 이동`으로 예약 가능 시간 영역에 바로 갈 수 있다.
- [ ] 예약 시간표는 한 번만 들어간 뒤 방향키로 이동할 수 있다.
- [ ] 대화상자를 닫으면 방금 열었던 예약 시간이나 현재 예약 내용으로 초점이 돌아온다.
- [ ] 조건 적용 후 결과 제목으로 초점이 이동한다.

## 5. 댓글 목록 점검표

### 5.1 공통

- [ ] 홈 화면의 `댓글 목록` 카드에서 댓글 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면에서 현재 댓글 상태와 목표 댓글이 보인다.
- [ ] 수행 탭 하단에는 `결과로 바로 이동`, `이 탭 닫기` 두 버튼만 보인다.

### 5.2 비교안 A

- [ ] 댓글마다 작성자, 작성 시각, 도움이 돼요, 답글 보기, 댓글 정보 보기 등이 각각 탭 이동 지점으로 노출된다.
- [ ] 댓글 정보 대화상자를 닫으면 원래 댓글 작업으로 돌아가지 않고 댓글 목록 제목 근처로 돌아온다.
- [ ] 과업 1에서 `최신순 + 후기 + 민지 댓글 답글 보기`가 수행된다.
- [ ] 과업 2에서 `도움이 많은 순 + 운영자 댓글 정보 보기 후 도움이 돼요`가 수행된다.
- [ ] 과업 3에서 `질문 댓글만 보기 + 준호 댓글 답글 보기`가 수행된다.

### 5.3 비교안 B

- [ ] `댓글 목록으로 바로 이동` 링크나 버튼으로 첫 진입 부담을 줄일 수 있다.
- [ ] 댓글은 하나의 선택 항목으로 제공되고 방향키로 이동할 수 있다.
- [ ] `선택한 댓글 작업` 영역에서 답글 보기, 댓글 정보 보기, 도움이 돼요를 이어서 수행할 수 있다.
- [ ] 댓글 정보 대화상자를 닫으면 방금 사용한 작업 버튼으로 초점이 돌아온다.
- [ ] 댓글 범위를 적용하면 댓글 목록 제목으로 초점이 이동한다.

## 6. 상품 옵션 선택 점검표

### 6.1 공통

- [ ] 홈 화면의 `상품 옵션 선택` 카드에서 상품 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면에서 현재 선택 상태와 목표 옵션 조합이 보인다.
- [ ] 수행 탭 하단에는 `옵션 영역으로 바로 이동`, `이 탭 닫기` 두 버튼만 보인다.

### 6.2 비교안 A

- [ ] 상단 보조 링크와 상품 안내 링크를 지나야 옵션 선택 묶음에 도달한다.
- [ ] 각 옵션 줄마다 `선택`, `옵션 설명 보기`가 따로 있어 순차 이동이 길다.
- [ ] 옵션 설명 대화상자를 닫으면 원래 옵션 줄 대신 `옵션 선택` 제목 근처로 돌아온다.
- [ ] 과업 1에서 `크림색 + 15형 + 충전기 주머니 포함 + 장바구니 담기`가 수행된다.
- [ ] 과업 2에서 `13형 설명 보기 후 먹색 기본형 담기`가 수행된다.
- [ ] 과업 3에서 `올리브 + 15형 + 문서 주머니 포함 + 수량 2개 담기`가 수행된다.

### 6.3 비교안 B

- [ ] `옵션 영역으로 바로 이동` 링크나 버튼으로 첫 진입 부담을 줄일 수 있다.
- [ ] 색상, 크기, 추가 구성은 한 번만 들어간 뒤 방향키로 이동할 수 있다.
- [ ] 각 옵션 묶음 아래에서 `선택한 ... 설명 보기` 버튼을 사용할 수 있다.
- [ ] 옵션 설명 대화상자를 닫으면 방금 사용한 설명 보기 버튼으로 초점이 돌아온다.
- [ ] `현재 선택과 장바구니` 영역에서 수량 변경과 장바구니 담기를 이어서 수행할 수 있다.

## 7. 결과와 저장 확인

- [ ] 각 서비스의 최종 비교 화면에서 결과 파일(JSON) 내려받기 링크가 동작한다.
- [ ] 사전 계산 기준과 실제 기록이 함께 표시된다.
- [ ] `docs` 폴더의 단계 보고서에 이번 변경 사항이 반영되어 있다.
- [ ] 설문지 주소를 넣었을 경우 결과 전달 링크가 올바르게 만들어진다.

- [ ] 브라우저 자동화로 검증하지 못한 항목은 이 문서의 체크 결과로 대체한다.
