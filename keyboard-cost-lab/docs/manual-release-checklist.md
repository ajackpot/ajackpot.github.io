# 배포 전 수동 점검표

부제: 브라우저 자동화의 한계를 보완하기 위한 서비스별 배포 점검표

## 1. 목적

이 문서는 브라우저 자동화로 끝까지 검증하기 어려운 항목을 사람이 직접 확인할 수 있도록 만든 배포용 점검표다. 현재 공개 범위는 `예약 캘린더`, `댓글 목록`, `검색 결과 목록` 3개 서비스이며, 과업 준비, 새 탭 수행, 과업 종료, 최종 결과 표시까지 확인한다.

## 2. 공통 준비

- [ ] `node scripts/run-benchmark.mjs`를 다시 실행해 사전 계산 결과 파일을 최신 상태로 만들었다.
- [ ] 정적 서버에서 `index.html`, `comments.html`, `search.html`이 열린다.
- [ ] 정적 서버에서 `data/benchmark-results-calendar.json`, `data/benchmark-results-comments.json`, `data/benchmark-results-search.json`이 열린다.
- [ ] 삭제한 서비스 파일(`product.html`, `settings.html`, `filters.html`, `checkout.html`)이 공개 목록에 남아 있지 않다.
- [ ] 새 탭 열기 기능이 브라우저의 팝업 차단에 막히지 않는다.
- [ ] `docs` 폴더에 이번 단계 보고서와 수동 점검표가 함께 남아 있다.

## 3. 공통 화면 흐름 점검

### 3.1 홈 화면

- [ ] 첫 화면에서 바로 과업 시작 버튼이 보이지 않는다.
- [ ] 서비스 유형을 먼저 고른 뒤 해당 서비스 화면으로 들어간다.
- [ ] 공개 서비스는 `예약 캘린더`, `댓글 목록`, `검색 결과 목록` 3개만 보인다.
- [ ] 서비스 카드 버튼은 `예약 캘린더 화면으로 이동`, `댓글 목록 화면으로 이동`, `검색 결과 목록 화면으로 이동`처럼 서비스 이름만 읽어도 서로 구분된다.
- [ ] 홈 화면 문구에 불필요한 영어가 남아 있지 않다.

### 3.2 서비스 화면

- [ ] 서비스 요약은 무엇을 수행하는지 정도만 설명하고, 사전 예상 시간이나 개선 목표를 먼저 제시하지 않는다.
- [ ] 두 화면이 자동으로 섞인 순서로 열린다는 안내가 있다.
- [ ] 결과는 두 화면을 모두 수행한 뒤 한 번에 표시된다는 안내가 있다.
- [ ] `과업 준비로 이동`, `서비스 선택으로 돌아가기` 문구가 동작을 분명하게 설명한다.
- [ ] 용어 설명 보기에서 남겨 둔 용어의 뜻을 확인할 수 있다.

### 3.3 과업 준비 화면

- [ ] 과업 준비 화면에는 `이번 요청`과 필수 진행 방법만 표시된다.
- [ ] 어떤 버튼을 몇 번 누르거나 어떤 순서로 탐색하라는 세부 지시가 없다.
- [ ] 사전 예상 시간, 목표 개선폭, 사용자 유형별 기준표가 과업 수행 전에 보이지 않는다.
- [ ] 수행 탭을 열기 전에도 과업 요청을 읽을 수 있고, 새 탭을 열어도 원래 창이 남아 있다.
- [ ] `과업을 모두 수행했다고 판단하면 수행 탭 하단의 과업 종료 버튼을 누르십시오.` 안내가 눈에 띄게 제공된다.
- [ ] 수행할 수 없다고 판단해도 `과업 종료` 버튼으로 다음 단계로 넘어갈 수 있다는 안내가 있다.

### 3.4 A/B 순서와 결과 표시

- [ ] 새 실험을 여러 번 시작하면 `A → B`와 `B → A` 순서가 모두 나올 수 있다.
- [ ] 첫 번째 화면을 모두 마쳐도 중간 결과 화면이 나오지 않고, 다음 과업이나 다음 화면으로 이어진다.
- [ ] 두 화면의 모든 과업을 끝낸 뒤에만 최종 결과 화면이 나온다.
- [ ] 최종 결과 화면에서 실제 기록, 수행 완료 기록, 수행 불가능 기록, 사전 계산 기준을 확인할 수 있다.
- [ ] 결과 파일(JSON) 내려받기 링크가 동작한다.

### 3.5 수행 탭 계측과 종료

- [ ] 수행 탭을 열자마자 시간 기록이 시작되지 않는다.
- [ ] 첫 키 입력이나 첫 클릭이 들어가는 순간부터 기록이 시작된다.
- [ ] 수행 탭을 다른 탭으로 가려 두었을 때 숨김 시간이 결과에서 빠진다.
- [ ] 수행 탭 하단에는 `...으로 이동` 버튼과 `과업 종료` 버튼만 보인다.
- [ ] 하단 보조 버튼 조작은 실제 지표에 포함되지 않는다.
- [ ] 목표 행동을 수행하면 서비스 화면 안의 최종 확인 대화상자가 열린다.
- [ ] 최종 확인 대화상자의 `확인`을 누른 뒤 `과업 종료` 버튼을 누르면 수행 완료로 기록된다.
- [ ] 목표 행동을 하지 않고 `과업 종료` 버튼을 누르면 수행 불가능 기록으로 넘어간다.
- [ ] 과업 종료 뒤 뜨는 기록 저장 대화상자에서 `확인`을 누르면 수행 탭이 닫히거나 원래 화면으로 돌아간다.

### 3.6 실제 서비스처럼 보이는 기능

- [ ] 수행 탭에는 서비스명, 검색, 알림, 보관, 내 상담, 신고 등 실제 사이트에 있을 법한 보조 기능이 보인다.
- [ ] 보조 기능을 눌렀을 때 페이지 위로 튀거나 실험 밖으로 나가지 않는다.
- [ ] 실행하지 않는 기능은 `현재 점검 중` 같은 안내를 보여 준다.
- [ ] 수행 탭 안에는 비교안 설명, 실험 의도, 사전 시간 계산 같은 비서비스 문구가 남아 있지 않다.

### 3.7 언어와 이해 가능성

- [ ] `slot`, `skip link`, `modal`, `focus` 같은 낯선 영어가 화면에 그대로 남아 있지 않다.
- [ ] 버튼과 링크 문구만 읽어도 눌렀을 때 일어나는 결과를 예상할 수 있다.
- [ ] 같은 뜻의 용어가 페이지마다 달라지지 않는다.
- [ ] 설명이 필요한 용어는 용어 설명 보기나 인접 문구에서 뜻을 확인할 수 있다.

## 4. 예약 캘린더 점검표

### 4.1 공통

- [ ] 홈 화면의 `예약 캘린더` 카드에서 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 너무 친절한 탐색 지시로 바뀌어 있지 않다.
- [ ] 수행 탭 하단에는 `예약 가능 시간으로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 상단의 상담 서비스 검색, 알림, 내 상담 같은 보조 기능은 안내를 띄우거나 화면 안에서만 반응한다.

### 4.2 비교안 A

- [ ] 상단 보조 기능과 조건 선택을 지나야 예약 가능 시간에 도달한다.
- [ ] 예약 시간마다 `선택`, `시간 안내 보기`가 따로 있어 순차 이동이 길다.
- [ ] 대화상자를 닫으면 방금 보던 예약 시간 대신 `예약 가능 시간` 제목 근처로 돌아온다.
- [ ] 목표 예약을 완료하고 최종 확인 대화상자를 확인한 뒤 `과업 종료`를 누르면 수행 완료로 기록된다.

### 4.3 비교안 B

- [ ] 맨 앞의 `예약 가능 시간으로 바로 이동` 링크로 예약 가능 시간 영역에 바로 갈 수 있다.
- [ ] 예약 시간표는 한 번만 들어간 뒤 방향키로 이동할 수 있다.
- [ ] 대화상자를 닫으면 방금 열었던 예약 시간이나 현재 예약 내용으로 초점이 돌아온다.
- [ ] 목표 예약을 완료하지 않고 `과업 종료`를 누르면 수행 불가능 기록으로 넘어간다.

## 5. 댓글 목록 점검표

### 5.1 공통

- [ ] 홈 화면의 `댓글 목록` 카드에서 댓글 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 댓글 제목이나 대상 정도만 제시하고, 구체적 조작 순서를 안내하지 않는다.
- [ ] 수행 탭 하단에는 `댓글 목록으로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 커뮤니티 보관, 신고, 알림 같은 보조 기능은 안내를 띄우거나 화면 안에서만 반응한다.

### 5.2 비교안 A

- [ ] 댓글마다 작성자, 작성 시각, 도움이 돼요, 답글 보기, 댓글 정보 보기 등이 각각 탭 이동 지점으로 노출된다.
- [ ] 댓글 정보 대화상자를 닫으면 원래 댓글 작업으로 돌아가지 않고 댓글 목록 제목 근처로 돌아온다.
- [ ] 목표 댓글의 답글 보기나 반응 남기기 뒤 최종 확인 대화상자가 열린다.
- [ ] 최종 확인 대화상자를 확인한 뒤 `과업 종료`를 누르면 수행 완료로 기록된다.

### 5.3 비교안 B

- [ ] 맨 앞의 `댓글 목록으로 바로 이동` 링크로 첫 진입 부담을 줄일 수 있다.
- [ ] 댓글은 하나의 선택 항목으로 제공되고 방향키로 이동할 수 있다.
- [ ] `선택한 댓글 작업` 영역에서 답글 보기, 댓글 정보 보기, 도움이 돼요를 이어서 수행할 수 있다.
- [ ] 목표 행동을 하지 않고 `과업 종료`를 누르면 수행 불가능 기록으로 넘어간다.

## 6. 검색 결과 목록 점검표

### 6.1 공통

- [ ] 홈 화면의 `검색 결과 목록` 카드에서 검색 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 자료명이나 처리할 일 정도만 제시하고, 탐색 방법을 안내하지 않는다.
- [ ] 수행 탭 하단에는 `검색 결과로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 자료실 검색, 검색 알림, 보관함 같은 보조 기능은 안내를 띄우거나 화면 안에서만 반응한다.

### 6.2 비교안 A

- [ ] 상단 보조 링크와 자료 조건 선택을 지나야 검색 결과에 도달한다.
- [ ] 자료마다 제목 링크, 갱신 시각, 공유, 저장, 바로 열기, 미리보기가 각각 따로 있어 순차 이동이 길다.
- [ ] 미리보기 대화상자를 닫으면 원래 자료 작업 대신 검색 결과 제목 근처로 돌아온다.
- [ ] 목표 자료의 미리보기, 저장, 바로 열기 뒤 최종 확인 대화상자가 열린다.

### 6.3 비교안 B

- [ ] 맨 앞의 `검색 결과로 바로 이동` 링크로 첫 진입 부담을 줄일 수 있다.
- [ ] 자료는 하나의 선택 항목으로 제공되고 방향키로 이동할 수 있다.
- [ ] `선택한 자료 작업` 영역에서 미리보기, 저장, 바로 열기를 이어서 수행할 수 있다.
- [ ] 목표 행동을 하지 않고 `과업 종료`를 누르면 수행 불가능 기록으로 넘어간다.

## 7. 배포 메모

- [ ] 이번 배포에서 공개하는 서비스 유형 3개를 README에 반영했다.
- [ ] 서비스 유형을 다시 늘릴 경우 홈 화면, 벤치마크 생성 목록, 수동 점검표, 용어 가이드를 함께 갱신한다.
- [ ] 공통 모듈을 바꾼 배포라면 `예약 캘린더`, `댓글 목록`, `검색 결과 목록` 3개 서비스에서 진입→과업 준비→새 탭 수행→최종 확인 대화상자→과업 종료→기록 저장 대화상자→최종 결과 흐름을 다시 확인한다.
- [ ] 브라우저 자동화로 검증하지 못한 항목은 이 문서의 체크 결과로 대체한다.
