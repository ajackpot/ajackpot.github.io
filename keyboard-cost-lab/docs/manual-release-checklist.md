# 배포 전 수동 점검표

부제: 브라우저 자동화의 한계를 보완하기 위한 서비스별 배포 점검표

## 1. 목적

이 문서는 브라우저 자동화로 끝까지 검증하기 어려운 항목을 사람이 직접 확인할 수 있도록 만든 배포용 점검표다. 현재 공개 범위는 `예약 캘린더`, `댓글 목록`, `검색 결과 목록` 3개 서비스이며, 과업 준비, 새 탭 수행, 과업 종료, 최종 결과 표시까지 확인한다.

## 2. 공통 준비

- [ ] `node scripts/run-benchmark.mjs`를 다시 실행해 사전 계산 결과 파일을 최신 상태로 만들었다.
- [ ] `node scripts/static-quality-check.mjs`를 실행해 내부 식별값 비노출, 수행 탭 요청 표시 옵션, 주요 실행 버튼 연결을 확인했다.
- [ ] 정적 서버에서 `index.html`, `comments.html`, `search.html`이 열린다.
- [ ] 정적 서버에서 `data/benchmark-results-calendar.json`, `data/benchmark-results-comments.json`, `data/benchmark-results-search.json`이 열린다.
- [ ] 삭제한 서비스 파일(`product.html`, `settings.html`, `filters.html`, `checkout.html`)이 공개 목록에 남아 있지 않다.
- [ ] 새 탭 열기 기능이 브라우저의 팝업 차단에 막히지 않는다.
- [ ] `docs` 폴더에 이번 단계 보고서와 수동 점검표가 함께 남아 있다.

## 3. 공통 화면 흐름 점검

### 3.1 홈 화면

- [ ] 첫 화면에서 바로 과업 시작 버튼이 보이지 않는다.
- [ ] 서비스 유형을 먼저 고른 뒤 해당 서비스 화면으로 들어간다.
- [ ] 공개 서비스는 `예약 캘린더`, `댓글 목록`, `검색 결과 목록` 3개만 보이고, 각 서비스는 2개 과업으로 진행된다.
- [ ] 서비스 카드 버튼은 `예약 캘린더 화면으로 이동`, `댓글 목록 화면으로 이동`, `검색 결과 목록 화면으로 이동`처럼 서비스 이름만 읽어도 서로 구분된다.
- [ ] 서비스 카드에는 과업 수, 비교 화면 수, 결과 표시 방식 같은 반복 안내 문구가 노출되지 않는다.
- [ ] 아직 기록이 없는 서비스는 진행 상태가 `수행 전`으로 표시된다.
- [ ] 일부 과업 기록만 저장된 서비스는 진행 상태가 `진행 중`으로 표시된다.
- [ ] 두 비교 화면의 과업 기록이 모두 저장된 서비스는 진행 상태가 `수행 완료`로 표시된다.
- [ ] 한 서비스 수행 뒤 홈으로 돌아와도 다른 서비스의 기존 수행 상태가 사라지지 않는다.
- [ ] 홈 화면 문구에 불필요한 영어가 남아 있지 않다.
- [ ] 홈 화면과 과업 준비 화면에 실험 번호 같은 내부 식별값이 표시되지 않는다.

### 3.2 서비스 화면

- [ ] 서비스 요약은 무엇을 수행하는지 정도만 설명하고, 사전 예상 시간이나 개선 목표를 먼저 제시하지 않는다.
- [ ] 두 화면이 자동으로 섞인 순서로 열린다는 안내가 있다.
- [ ] 결과는 두 화면을 모두 수행한 뒤 한 번에 표시된다는 안내가 있다.
- [ ] `과업 준비로 이동`, `서비스 선택으로 돌아가기` 문구가 동작을 분명하게 설명한다.
- [ ] 용어 설명 보기에서 남겨 둔 용어의 뜻을 확인할 수 있다.

### 3.3 과업 준비 화면

- [ ] 과업 준비 화면에는 `이번 요청`과 필수 진행 방법만 표시된다.
- [ ] 어떤 버튼을 몇 번 누르거나 어떤 순서로 탐색하라는 세부 지시가 없다.
- [ ] 사전 예상 시간, 목표 개선폭, 사용자 유형별 기준표가 과업 수행 전에 보이지 않는다.
- [ ] 수행 탭을 열기 전에도 과업 요청을 읽을 수 있고, 새 탭을 열어도 원래 창이 남아 있다.
- [ ] `과업 요청 사항을 수행 탭에서도 보기` 스위치를 켜면 수행 탭 맨 위에 이번 요청이 표시된다.
- [ ] `과업 요청 사항을 수행 탭에서도 보기` 스위치를 끄면 수행 탭에는 실제 서비스 화면이 먼저 표시된다.
- [ ] `과업을 모두 수행했다고 판단하면 수행 탭 하단의 과업 종료 버튼을 누르십시오.` 안내가 눈에 띄게 제공된다.
- [ ] `수행할 수 없다고 판단해도 최하단의 과업 종료 버튼을 실행하여 다음 단계로 건너뛸 수 있습니다.`라는 안내가 있다.

### 3.4 A/B 순서와 결과 표시

- [ ] 새 실험을 여러 번 시작하면 `A → B`와 `B → A` 순서가 모두 나올 수 있다.
- [ ] 첫 번째 화면을 모두 마쳐도 중간 결과 화면이 나오지 않고, 다음 과업이나 다음 화면으로 이어진다.
- [ ] 두 화면의 모든 과업을 끝낸 뒤에만 최종 결과 화면이 나온다.
- [ ] 최종 결과 화면에서 실제 기록, 수행 완료 기록, 수행 불가능 기록, 사전 계산 기준을 확인할 수 있다.
- [ ] 결과 파일(JSON) 내려받기 링크가 동작한다.

### 3.5 수행 탭 계측과 종료

- [ ] 수행 탭을 열자마자 시간 기록이 시작되지 않는다.
- [ ] 첫 키 입력이나 첫 클릭이 들어가는 순간부터 기록이 시작된다.
- [ ] 수행 탭을 다른 탭으로 가려 두었을 때 숨김 시간이 결과에서 빠진다.
- [ ] 수행 탭 하단에는 `...으로 이동` 버튼과 `과업 종료` 버튼만 보인다.
- [ ] 하단 보조 버튼 조작은 실제 지표에 포함되지 않는다.
- [ ] 수행 중에는 성공·실패·정답 여부를 암시하는 화면 메시지나 화면낭독용 상태 메시지가 나오지 않는다.
- [ ] 목표 행동을 수행하면 실제 서비스 동작처럼 보이는 예약 완료 화면, 답글 목록 질문, 토스트, 자료 처리 안내 등이 표시된다.
- [ ] 서비스 피드백은 작업 접수, 표시, 저장, 닫기 같은 사실만 알리고 과업 성공 여부는 말하지 않는다.
- [ ] `과업 종료` 버튼을 누르면 바로 종료되지 않고, `과업을 종료하시겠습니까?` 확인 대화상자가 먼저 열린다.
- [ ] 종료 확인 대화상자에서 `아니요, 계속합니다`를 누르면 과업 수행 화면으로 돌아간다.
- [ ] 종료 확인 대화상자는 성공·실패 여부를 먼저 알려 주지 않고, 종료 여부만 묻는다.
- [ ] 종료 확인 대화상자에서 `예, 종료합니다`를 누른 뒤에만 기록 저장 대화상자에서 `과업 수행에 성공했습니다.` 또는 실패 사유 메시지가 표시된다.
- [ ] 목표 행동을 하지 않고 `예, 종료합니다`를 누르면 수행 불가능 기록 또는 실패 사유로 넘어간다.
- [ ] 과업 종료 확인과 기록 저장 대화상자 조작은 실제 지표에 포함되지 않는다.
- [ ] 과업 종료 뒤 뜨는 기록 저장 대화상자에서 `확인`을 누르면 수행 탭이 닫히거나 원래 화면으로 돌아간다.

### 3.6 실제 서비스처럼 보이는 기능

- [ ] 수행 탭에는 서비스명, 검색, 알림, 보관, 내 상담, 신고 등 실제 사이트에 있을 법한 보조 기능이 보인다.
- [ ] 보조 기능을 눌렀을 때 페이지 위로 튀거나 실험 밖으로 나가지 않는다.
- [ ] 간단한 보조 기능은 가능하면 화면 안 패널, 상태 변경, 조건 입력, 안내 표처럼 실제 기능에 가깝게 반응한다.
- [ ] 구현 범위가 큰 기능만 `현재 점검 중` 같은 안내를 보여 준다.
- [ ] 수행 탭 안에는 비교안 설명, 실험 의도, 사전 시간 계산 같은 비서비스 문구가 남아 있지 않다.

### 3.7 언어와 이해 가능성

- [ ] `slot`, `skip link`, `modal`, `focus` 같은 낯선 영어가 화면에 그대로 남아 있지 않다.
- [ ] 버튼과 링크 문구만 읽어도 눌렀을 때 일어나는 결과를 예상할 수 있다.
- [ ] 같은 뜻의 용어가 페이지마다 달라지지 않는다.
- [ ] 설명이 필요한 용어는 용어 설명 보기나 인접 문구에서 뜻을 확인할 수 있다.

## 4. 예약 캘린더 점검표

### 4.1 공통

- [ ] 홈 화면의 `예약 캘린더` 카드에서 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 너무 친절한 탐색 지시로 바뀌어 있지 않다.
- [ ] 수행 탭 하단에는 `예약 가능 시간으로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 상단의 상담 서비스 검색, 알림, 내 상담 같은 보조 기능은 페이지 안 패널이나 상태 변경으로 반응한다.
- [ ] 상담 서비스 검색 입력란에서 Enter를 누르면 페이지 밖으로 이동하지 않고 검색 결과 패널이 열린다.
- [ ] `처음 화면`, `상담사 소개`, `이용권`, `이용 후기`, `가격 안내`, `자주 묻는 질문`, `운영 정책`, `문의`가 각각 화면 안에서 탐색 가능한 패널을 연다.
- [ ] 전체 메뉴는 기본적으로 접혀 있으며, 펼치기/접기 버튼의 상태가 초점 이동과 화면낭독으로 이해된다.
- [ ] 비교안 A에서는 메뉴가 접혀 있어도 숨겨진 보조 메뉴가 여전히 Tab 이동 대상이 되어 과도한 탐색 부담을 재현한다.
- [ ] 비교안 B에서는 접힌 메뉴가 Tab 이동 대상에서 빠진다.
- [ ] 상담사 소개의 `이 상담사 일정 보기`, 검색 결과의 `비대면 조건 입력` 같은 버튼은 조건 입력 상태를 바꾸고 조건 적용 버튼으로 초점을 보낸다.
- [ ] 알림 설정, 이용권 선택, 후기 보관, 문의 접수 같은 간단한 상태 변경 기능은 중립적인 상태 안내를 표시한다.
- [ ] 실시간 채팅처럼 구현 범위가 큰 기능만 점검 중 안내를 표시한다.

### 4.2 비교안 A

- [ ] 상단 보조 기능과 조건 선택을 지나야 예약 가능 시간에 도달한다.
- [ ] 예약 시간마다 `선택`, `시간 안내 보기`가 따로 있어 순차 이동이 길다.
- [ ] 대화상자를 닫으면 방금 보던 예약 시간 대신 `예약 가능 시간` 제목 근처로 돌아온다.
- [ ] 예약을 확정해도 수행 중에는 정답 여부가 표시되지 않는다.
- [ ] 김민아 상담사 화요일 14:30 비대면 30분 예약에서는 예약 확정 전에 상담 주제, 사전 작성, 알림 방법을 선택하는 화면이 열린다.
- [ ] 비교안 A의 상담 옵션 선택은 시각적으로는 접힌 선택값처럼 보이지만 실제로는 여러 선택지가 버튼처럼 탐색되어 조작 부담을 만든다.
- [ ] 상담 옵션을 잘못 고르거나 선택하지 않은 채 종료하면 최종 결과에서 실패 사유로 기록된다.
- [ ] 박하늘 상담사 목요일 10:00 대면 30분 예약 과업은 상담 옵션 없이 예약이 이어진다.
- [ ] 한 주 최대 2개 예약 상태에서 새 예약을 시도하면 기존 예약을 먼저 취소하라는 안내 대화상자가 열린다.
- [ ] 기존 예약 2개 중 하나를 취소한 뒤 박하늘 상담사의 목요일 10:00 대면 30분 예약을 완료하면 성공으로 기록된다.
- [ ] 예약 확정 화면을 확인한 뒤 `과업 종료` → `예, 종료합니다`를 누르면 최종 결과 메시지가 표시된다.

### 4.3 비교안 B

- [ ] 맨 앞의 `예약 가능 시간으로 바로 이동` 링크로 예약 가능 시간 영역에 바로 갈 수 있다.
- [ ] 예약 시간표는 한 번만 들어간 뒤 방향키로 이동할 수 있다.
- [ ] 예약 시간표의 각 항목은 Enter, Space, 마우스 클릭으로 모두 예약 확인 대화상자를 연다.
- [ ] 대화상자를 닫으면 방금 열었던 예약 시간이나 현재 예약 내용으로 초점이 돌아온다.
- [ ] 예약 캘린더 B가 첫 번째 화면 또는 두 번째 화면으로 열려도 예약 가능 시간 콘텐츠가 탐색된다.
- [ ] 목표 예약을 완료하지 않고 `과업 종료` → `예, 종료합니다`를 누르면 수행 불가능 기록 또는 실패 사유로 넘어간다.

## 5. 댓글 목록 점검표

### 5.1 공통

- [ ] 홈 화면의 `댓글 목록` 카드에서 댓글 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 댓글 제목이나 대상 정도만 제시하고, 구체적 조작 순서를 안내하지 않는다.
- [ ] 일반 댓글 과업의 수행 탭 하단에는 `댓글 목록으로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 답글 질문 과업의 수행 탭 하단에는 질문과 답변 선택 콤보상자가 `답변 제출하고 과업 종료하기` 버튼 위에 보인다.
- [ ] 답글 질문 과업에서 답글 내용은 고정되어 보이지만 답글 작성자 이름과 정답 선택지는 수행 화면마다 배정된 이름 기준으로 표시된다.
- [ ] 한 비교안에서 본 답을 다른 비교안에서 답글 목록에 들어가지 않고 바로 제출해도, 목표 답글 목록 진입 여부가 없으면 성공으로 기록되지 않는다.
- [ ] 답글 질문 과업에서 답글 본문과 작성 시간은 유지되고, 작성자 이름과 하단 선택지는 수행 화면별 배정값을 따른다.
- [ ] A에서 확인한 작성자 이름을 B에서 답글 목록 진입 없이 제출해도, 지정 답글 목록 진입 기록이 없으면 성공으로 기록되지 않는다.
- [ ] 커뮤니티 보관, 신고, 알림, 내 댓글, 작성 안내 같은 보조 기능은 점검 중 안내만 반복하지 않고 화면 안 패널, 토스트, 상태 변경처럼 반응한다.

### 5.2 비교안 A

- [ ] 댓글마다 작성자, 작성 시각, 도움이 돼요, 답글 보기, 댓글 정보 보기 등이 각각 탭 이동 지점으로 노출된다.
- [ ] 조건 선택 콤보상자는 비교안 A에서 접근은 가능하지만 여러 선택지가 버튼처럼 장황하게 탐색되는 변형으로 제공된다.
- [ ] 조건을 올바르게 적용했는지는 수행 중 알려 주지 않지만 내부 판정에는 반영된다.
- [ ] 댓글 정보 대화상자를 닫으면 원래 댓글 작업으로 돌아가지 않고 댓글 목록 제목 근처로 돌아온다.
- [ ] 답글 보기 뒤에는 실제 답글 목록이 표시되고, 요청된 답글 정보는 하단 과업 종료 영역의 객관식 콤보상자에서 선택한 뒤 한 번에 종료할 수 있다.
- [ ] 답글 질문 과업의 작성자 이름은 A/B 화면마다 현재 노출된 답글 목록 기준으로 표시되고, 하단 답변 선택지도 같은 이름 배정을 따른다.
- [ ] 첫 번째 화면에서 본 답을 기억해 두 번째 화면에서 답글 목록을 열지 않고 제출해도 성공으로 처리되지 않는다.
- [ ] `도움이 돼요`를 누르면 화면 하단 토스트가 표시되고 화면낭독기가 읽을 수 있다.
- [ ] 댓글 작업 뒤에도 수행 중에는 정답 여부가 표시되지 않는다.
- [ ] `과업 종료` → `예, 종료합니다`를 누른 뒤에만 최종 결과 메시지가 표시된다.

### 5.3 비교안 B

- [ ] 맨 앞의 `댓글 목록으로 바로 이동` 링크로 첫 진입 부담을 줄일 수 있다.
- [ ] 댓글은 하나의 선택 항목으로 제공되고 방향키로 이동할 수 있다.
- [ ] `선택한 댓글 작업` 영역에서 답글 보기, 댓글 정보 보기, 도움이 돼요를 이어서 수행할 수 있다.
- [ ] 목표 행동을 하지 않고 `과업 종료` → `예, 종료합니다`를 누르면 수행 불가능 기록 또는 실패 사유로 넘어간다.

## 6. 검색 결과 목록 점검표

### 6.1 공통

- [ ] 홈 화면의 `검색 결과 목록` 카드에서 검색 서비스 화면으로 들어갈 수 있다.
- [ ] 과업 준비 화면의 요청이 자료명이나 처리할 일 정도만 제시하고, 탐색 방법을 안내하지 않는다.
- [ ] 수행 탭 하단에는 `검색 결과로 이동`, `과업 종료` 두 버튼만 보인다.
- [ ] 자료실 검색, 검색 알림, 보관함, 검색 기록, 자료 요청, 이용 안내, 문의 같은 보조 기능은 화면 안 패널, 토스트, 상태 변경으로 반응한다.
- [ ] 검색 보조 기능 패널은 목표 자료나 정답 여부를 직접 알려 주지 않는다.
- [ ] 비교안 B 최상단에는 `검색 결과 영역으로 바로가기`, `검색 조건 설정으로 바로가기` 링크가 제공된다.
- [ ] 검색 도움말, 자료 범위 설명, 검색 기록, 보관함, 자료 요청, 문의, 검색 알림 패널은 단순 점검 중 안내가 아니라 실제 설명·상태 변경·토스트처럼 반응한다.

### 6.2 비교안 A

- [ ] 상단 보조 링크와 자료 조건 선택을 지나야 검색 결과에 도달한다.
- [ ] 자료마다 제목 링크, 갱신 시각, 공유, 저장, 바로 열기, 미리보기가 각각 따로 있어 순차 이동이 길다.
- [ ] 미리보기 대화상자를 닫으면 원래 자료 작업 대신 검색 결과 제목 근처로 돌아온다.
- [ ] 목표 자료 저장과 바로 열기는 중립적인 토스트 알림으로 처리된다.
- [ ] 미리보기 대화상자를 닫는 과업은 별도 안내 메시지 없이 원래 흐름으로 돌아온다.
- [ ] 예약 변경 안내 미리보기 과업은 하단 과업 종료 영역의 객관식 답변 선택과 함께 종료된다.
- [ ] 미리보기를 실제로 열지 않고 답만 고른 경우 성공으로 기록되지 않는다.
- [ ] 비대면 상담 연결 방법 저장 과업은 저장 위치, 포함할 내용, 저장 형식 옵션을 선택한 뒤 저장된다.
- [ ] 비교안 A의 저장 옵션 선택은 장황한 버튼형 선택지로 탐색 부담을 높인다.
- [ ] 자료 작업 뒤에도 수행 중에는 정답 여부가 표시되지 않는다.

### 6.3 비교안 B

- [ ] 맨 앞의 `검색 결과로 바로 이동` 링크로 첫 진입 부담을 줄일 수 있다.
- [ ] 자료는 하나의 선택 항목으로 제공되고 방향키로 이동할 수 있다.
- [ ] `선택한 자료 작업` 영역에서 미리보기, 저장, 바로 열기를 이어서 수행할 수 있다.
- [ ] 목표 행동을 하지 않고 `과업 종료` → `예, 종료합니다`를 누르면 수행 불가능 기록 또는 실패 사유로 넘어간다.
