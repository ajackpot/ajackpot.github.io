# 21단계 보고서: 19/20단계 병합 확인과 서비스별 진행 상태 저장

## 1. 작업 배경

최근 압축본에서 19단계 변경이 빠진 것처럼 보일 수 있다는 피드백이 있었다. 따라서 20단계 압축본을 최신 기준으로 다시 풀고, 19단계에서 들어간 댓글 답변 종료 영역 통합, 검색 결과 토스트 처리, 댓글 보조 기능 강화가 유지되는지 먼저 확인했다. 이후 홈 화면에서 반복 노출되던 `3개 과업`, `비교 화면 2개`, `수행 뒤 한 번에 결과 확인` 안내를 제거하고, 그 자리에 서비스별 진행 상태를 표시하도록 변경했다.

이 단계의 기준은 실험 제공자의 프레임이나 설명을 과도하게 노출하지 않고, 사용자의 실제 수행 상태와 기록 보존 여부를 명확히 관리하는 것이다.

## 2. 병합 기준

- 기준 압축본: `keyboard-cost-lab-step-20(1).zip`
- 병합 확인 대상: `keyboard-cost-lab-step-19(1).zip`에서 들어간 핵심 기능
- 확인한 19단계 기능
  - 댓글 답글 질문 과업에서 하단 과업 종료 영역에 답변 선택 콤보상자를 표시한다.
  - 답변이 필요한 과업의 종료 버튼은 `답변 제출하고 과업 종료하기`로 표시된다.
  - 검색 결과 목록의 저장과 바로 열기는 처리 결과 확인 팝업 대신 토스트 성격의 상태 알림으로 처리한다.
  - 검색 결과 미리보기 닫기는 별도 완료 안내 없이 원래 흐름으로 돌아간다.
  - 댓글 목록 보조 메뉴는 단순 점검 중 안내가 아니라 패널, 토스트, 상태 변경처럼 반응한다.
- 확인한 20단계 기능
  - 댓글 답글 작성자 이름은 수행 탭별로 무작위 배정된다.
  - 현재 수행 탭에 표시된 작성자 이름을 기준으로 하단 답변 선택지와 정답 판정이 계산된다.
  - 검색 결과 목록의 보조 기능은 실제 서비스처럼 화면 안 패널과 상태 변경으로 반응한다.

## 3. 홈 화면 변경

기존 서비스 카드의 반복 안내 목록을 제거했다.

제거한 문구:

- `3개 과업`
- `비교 화면 2개`
- `수행 뒤 한 번에 결과 확인`

새로 표시하는 내용:

- 서비스 이름
- 서비스 요약
- 진행 상태
- 저장 상태
- 서비스 화면으로 이동 버튼

진행 상태는 다음 값으로 표시된다.

- `수행 전`: 저장된 과업 기록이 없다.
- `진행 중`: 일부 과업 기록이 저장되어 있다.
- `수행 완료`: 두 비교 화면의 모든 과업 기록이 저장되어 있다.

## 4. 진행 상태 저장 구조

새 공통 모듈 `lib/experiment-store.js`를 추가했다.

브라우저 저장소 키:

```text
keyboard-cost-lab-results-v1
```

저장되는 주요 값:

- 서비스 ID
- 서비스 이름
- 서비스별 세션 ID
- A/B 수행 순서
- 서비스별 과업 수
- 비교 화면 수
- 계측 규칙
- 비교안 A 실제 과업 기록
- 비교안 B 실제 과업 기록
- 비교안별 실제 합계
- 사전 계산 요약
- 상태값과 상태 라벨
- 마지막 저장 시각

각 서비스는 과업 수행 탭에서 기록이 전달될 때마다 현재 서비스의 전체 과업 기록을 저장한다. 따라서 사용자가 한 서비스를 마친 뒤 다른 서비스로 이동해도 이전 서비스의 기록은 브라우저 저장소 안에 유지된다.

## 5. 결과 파일 변경

기존 결과 파일은 현재 서비스의 실제 기록과 사전 계산 기준을 중심으로 내보냈다. 이번 단계부터는 결과 파일의 `storedServices` 항목에 브라우저에 저장된 전체 서비스 기록을 함께 포함한다. 이 구조는 이후 설문지 연결 시 여러 서비스 기록을 한 번에 전달하기 위한 기반이다.

## 6. 수정 파일

- `app.js`
  - 홈 화면 서비스 카드의 반복 안내 목록 제거
  - 서비스별 진행 상태 표시
  - 예약 캘린더 과업 기록 저장소 연결
  - 결과 파일에 전체 서비스 저장값 포함
- `comments-app.js`
  - 댓글 목록 과업 기록 저장소 연결
  - 결과 파일에 전체 서비스 저장값 포함
- `search-app.js`
  - 검색 결과 목록 과업 기록 저장소 연결
  - 결과 파일에 전체 서비스 저장값 포함
- `data/service-registry.js`
  - 서비스 카드 반복 안내 배열 제거
  - 서비스별 과업 수와 비교 화면 수 추가
- `lib/service-shell.js`
  - 결과 파일 생성 시 `storedServices` 포함 가능하도록 확장
- `lib/experiment-store.js`
  - 서비스별 진행 상태와 실제 기록 저장/읽기 공통 모듈 추가
- `scripts/check-static-contracts.mjs`
  - 홈 반복 안내 제거, 진행 상태 표시, 저장소 연결, 전체 서비스 결과 내보내기 점검 추가
- `README.md`
  - 진행 상태 저장 구조와 새 파일 설명 반영
- `docs/manual-release-checklist.md`
  - 홈 화면 진행 상태와 결과 파일 전체 저장값 점검 항목 추가

## 7. 검증 결과

실행한 명령:

```bash
node --check app.js
node --check comments-app.js
node --check search-app.js
node --check lib/experiment-store.js
node --check lib/service-shell.js
node --check data/service-registry.js
node --check scripts/check-static-contracts.mjs
node scripts/run-benchmark.mjs
npm run check
```

추가 확인:

```bash
grep -R "3개 과업\|비교 화면 2개\|수행 뒤 한 번에 결과 확인\|service.points" -n app.js comments-app.js search-app.js data/service-registry.js lib
```

위 검색에서 사용자 화면용 홈 카드와 서비스 목록 코드에는 해당 반복 안내 문구가 남아 있지 않음을 확인했다.

정적 서버 응답 확인:

```text
index.html 200
comments.html 200
search.html 200
data/benchmark-results-calendar.json 200
data/benchmark-results-comments.json 200
data/benchmark-results-search.json 200
```

## 8. 남은 수동 점검

브라우저에서 실제로 확인할 항목은 다음이다.

- 처음 홈 화면에서 세 서비스 모두 `수행 전`으로 보이는지
- 한 서비스의 일부 과업을 저장한 뒤 홈으로 돌아오면 해당 서비스가 `진행 중`으로 보이는지
- 한 서비스의 두 비교 화면 과업을 모두 저장한 뒤 홈으로 돌아오면 해당 서비스가 `수행 완료`로 보이는지
- 다른 서비스로 이동해도 이전 서비스의 상태가 유지되는지
- 결과 파일을 내려받았을 때 `storedServices` 안에 여러 서비스 기록이 함께 들어가는지
