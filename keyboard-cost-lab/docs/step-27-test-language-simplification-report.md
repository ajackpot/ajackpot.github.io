# Step 27 · 테스트 용어 통일과 화면 설명 정리

## 목적

테스트 페이지에서 설문 대상자에게 불필요하게 길거나 기술적으로 보일 수 있는 설명을 줄였다. 특히 `실험`이라는 명칭을 사용자 화면에서는 `테스트`로 통일하고, 홈 화면과 서비스 진입 화면, 과업 준비 화면, 최종 비교 화면의 설명을 실제 수행 흐름에 맞게 정리했다.

## 주요 변경

- 홈 화면 제목과 문구를 `과도한 키보드 조작 테스트`, `테스트 시작 준비`, `테스트할 서비스 유형을 고르십시오.`로 바꾸었다.
- 홈 화면의 `현재 공개 범위`, 서비스명 나열 영역, `진행 상태 안내`, 브라우저 저장 설명을 삭제했다.
- 서비스 카드에서 `구현 완료` 상태 문구를 표시하지 않도록 했다.
- 설문지 제출 준비 안내를 `3개 서비스 과업을 모두 완료하면 설문지로 이동하여 수행 기록을 자동으로 입력할 수 있습니다.` 한 문장으로 줄였다.
- 서비스 진입 화면의 진행 방법에 `내용이 동일하고 이동 방식이 다른 두 개의 화면을 무작위 순서로 테스트 및 비교합니다.` 문장을 넣고, 기존의 `두 개의 화면은 자동으로 섞인 순서로 열립니다.` 문장을 삭제했다.
- 과업 준비 화면 설명을 `아래의 요청을 확인하고 (서비스 유형) 과업 수행 페이지를 여십시오.` 형식으로 바꾸었다.
- 최종 비교 화면 설명을 실제 기록과 사전 예상 기준 비교, 서비스 탐색 점검과 개선 기준 마련 용도로 나누어 표시했다.
- 별도 용어 설명 카드와 `비교안 A/B`, `초점`, `대화상자`, `결과 파일(JSON)` 용어 풀이를 사용자 화면에서 제거했다.
- 결과 파일 설명은 최종 비교 화면의 내려받기 안내에 통합했다.
- 사용자 화면과 현재 README, 수동 점검표에서 `실험` 표현을 `테스트` 표현으로 정리했다.
