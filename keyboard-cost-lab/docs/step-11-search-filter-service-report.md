# 11단계 보고서

부제: 검색 세부 조건 서비스 유형 추가와 사전 계산 기준 확장

## 1. 이번 단계 목표

이번 단계에서는 `검색 세부 조건` 서비스 유형을 기존 실험 틀에 맞춰 추가했다. 목표는 다음과 같았다.

- 홈 화면에서 `검색 세부 조건`을 구현 완료 상태로 선택할 수 있게 한다.
- 메인 창에서 과업을 확인하고, 새 탭에서 비교안 A/B를 수행하는 기존 흐름을 그대로 유지한다.
- 기간, 자료 종류, 담당 부서, 대상, 첨부 조건을 포함한 과업 3개를 만든다.
- 검색 세부 조건용 사전 계산 그래프와 결과 파일을 추가한다.
- 용어 가이드, 수동 점검표, README를 새 서비스 기준으로 갱신한다.

## 2. 구현한 내용

### 2.1 새 서비스 유형 추가

홈 화면에서 `검색 세부 조건` 카드를 선택할 수 있도록 `app.js`의 서비스 목록과 이동 분기를 갱신했다.

- 서비스 이름: `검색 세부 조건`
- 서비스 요약: 기간, 자료 종류, 담당 부서, 대상, 첨부 조건을 맞춘 뒤 원하는 자료 찾기
- 진입 파일: `filters.html`
- 실행 파일: `filters-app.js`

### 2.2 검색 세부 조건 시나리오와 과업

새 데이터 파일을 추가했다.

- `data/filters-scenario.js`
- `data/tasks-filters.js`
- `data/benchmark-graphs-filters.js`

세부 조건은 다음 다섯 가지로 구성했다.

- 기간
- 자료 종류
- 담당 부서
- 대상
- 첨부 있는 자료만 보기

과업은 다음 세 가지다.

1. 최근 7일 안내문에서 예약 변경 안내 미리보기 열었다가 닫기
2. 첨부 있는 질문답변에서 비대면 상담 연결 방법 저장
3. 비대면 상담 신청 서식에서 상담 준비 체크 목록 미리보기 후 바로 열기

### 2.3 비교안 A/B 구조

비교안 A는 세부 조건 묶음과 자료 작업이 모두 흩어져 있는 구조로 만들었다.

- 상단 보조 링크와 도움말 링크 뒤에 세부 조건 묶음 배치
- 기간, 자료 종류, 담당 부서, 대상, 첨부 조건을 각각 여러 개의 개별 버튼으로 제공
- 조건 적용 뒤 자료 카드마다 저장, 바로 열기, 미리보기 버튼이 따로 노출
- 미리보기 대화상자를 닫으면 검색 결과 제목 근처로 복귀

비교안 B는 빠른 이동과 묶음형 조건 선택을 통해 반복 이동 비용을 줄이는 구조로 만들었다.

- `세부 조건으로 바로 이동` 링크 제공
- 기간, 자료 종류, 담당 부서, 대상은 선택 목록으로 제공
- 첨부 조건은 같은 영역 안의 체크박스로 제공
- 자료 목록은 한 번만 들어간 뒤 방향키로 고를 수 있게 구성
- `선택한 자료 작업` 영역에서 미리보기, 저장, 바로 열기를 이어서 수행
- 미리보기 대화상자를 닫으면 같은 작업 버튼으로 초점 복귀

### 2.4 계측과 완료 처리

기존 서비스들과 같은 운영 원칙을 그대로 유지했다.

- 실제 계측은 수행 탭 첫 조작부터 시작
- 숨김 탭 시간 제외
- 수행 탭 하단 보조 버튼 영역은 측정 제외
- 과업 성공 뒤 완료 안내 팝업 표시
- `확인` 버튼으로 수행 탭 자동 닫기
