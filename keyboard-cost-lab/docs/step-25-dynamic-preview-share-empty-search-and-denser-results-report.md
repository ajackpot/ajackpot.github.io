# 25단계 보고서: 검색 미리보기 정답 무작위화, 예약 공유, 검색 기본값 제거, 목록 데이터 누적

## 배경

이번 단계는 A/B를 모두 수행하는 구조에서 첫 화면에서 얻은 답을 두 번째 화면에 그대로 제출하는 노이즈를 줄이고, 댓글과 검색 결과 목록에서 반복 항목 탐색 부담이 더 분명하게 드러나도록 하는 데 초점을 두었다.

## 변경 요약

### 1. 검색 결과 목록 미리보기 답변 무작위화

- `예약 변경 안내` 미리보기 과업의 핵심 숫자를 수행 화면별로 무작위 배정하도록 변경했다.
- 후보 값은 `4시간 전까지`, `6시간 전까지`, `8시간 전까지`, `12시간 전까지`이다.
- 미리보기 본문에 표시되는 숫자와 하단 답변 선택 콤보상자의 정답이 같은 내부 배정값을 사용한다.
- 가능하면 먼저 수행한 비교안의 정답값과 다음 비교안의 정답값이 같아지지 않도록 했다.
- 정답 선택지만 맞히는 것을 막기 위해 기존처럼 목표 자료의 미리보기를 실제로 열고 닫은 기록이 있어야 성공으로 기록된다.

수정 파일:

- `search-app.js`
- `data/tasks-search.js`
- `data/search-scenario.js`

### 2. 예약 캘린더 확인 대화상자 공유 기능 추가

- 예약 시간 확인 대화상자와 최종 예약 확인 대화상자에 `공유하기` 버튼을 추가했다.
- 공유하기는 실제 페이지 이동 없이 `선택한 예약 시간을 공유할 수 있는 주소를 복사했습니다.` 또는 `선택한 예약 정보를 공유할 수 있는 주소를 복사했습니다.` 토스트로 반응한다.
- 예약 성공 여부를 알려 주는 힌트는 추가하지 않았다.

수정 파일:

- `app.js`

### 3. 검색 결과 검색어 입력 기본값 제거

- 검색 결과 서비스의 검색어 입력란에 기본으로 보이던 `예약 변경` 값을 제거했다.
- 초기 검색어는 빈 값이며, 화면 요약에는 `검색어 없음`으로 표시된다.
- 검색 버튼을 빈 값으로 실행하면 `검색어 없이 전체 자료를 새로 고쳤습니다.`라는 중립적인 토스트를 표시한다.

수정 파일:

- `data/search-scenario.js`
- `search-app.js`
- `scripts/smoke-render-runner.mjs`

### 4. 댓글 목록 데이터 누적

반복 댓글 탐색 부담이 더 드러나도록 실제 커뮤니티에 있을 법한 댓글을 추가했다.

- 최신 후기 필터에서 `민지` 댓글 앞에 `예리`, `태민`, `지우` 후기 댓글이 나오도록 했다.
- 도움이 많은 순 필터에서 `운영자` 목표 댓글 앞에 도움이 수가 더 많은 운영자 안내 댓글 2개가 나오도록 했다.
- 비교안 B에서도 필터 적용 후 첫 항목이 목표 항목으로 바로 고정되지 않도록, 조건 적용 시 현재 선택 댓글을 첫 표시 항목으로 초기화한다.

수정 파일:

- `data/comments-scenario.js`
- `comments-app.js`
- `data/benchmark-graphs-comments.js`

### 5. 검색 결과 목록 데이터 누적

자료 목록에서도 목표 자료가 조건 적용 직후 첫 번째로 바로 접근되는 경향을 줄였다.

- 최신 안내문 필터에서 `예약 변경 안내` 앞에 실제 있을 법한 안내문 3개를 추가했다.
- 제목순 질문답변 필터에서 `비대면 상담 연결 방법` 앞에 계정, 대기실, 마이크 권한 관련 질문답변이 나오도록 했다.
- 비교안 B에서도 조건 적용 후 목표 자료가 기존 선택 상태로 유지되지 않도록, 조건 적용 시 현재 선택 자료를 첫 표시 자료로 초기화한다.

수정 파일:

- `data/search-scenario.js`
- `search-app.js`
- `data/benchmark-graphs-search.js`

## 남은 수동 점검

- 검색 결과 목록에서 `예약 변경 안내` 미리보기를 열었을 때 숫자가 수행 화면별로 달라질 수 있는지 확인한다.
- 하단 답변 선택지가 현재 미리보기 숫자와 맞는지 확인한다.
- 검색어 입력란이 처음부터 비어 있는지 확인한다.
- 댓글 최신 후기/운영자 안내 과업에서 목표 댓글이 필터 직후 첫 번째가 아닌지 확인한다.
- 검색 결과 최신 안내문/제목순 질문답변 과업에서 목표 자료가 필터 직후 첫 번째가 아닌지 확인한다.
- 예약 시간 확인과 최종 예약 확인 대화상자에서 `공유하기` 버튼이 중립 토스트로 반응하는지 확인한다.
