# 22단계 보고서: 서비스별 2개 과업 재구성 및 실제 서비스 흐름 강화

## 1. 작업 목적

이번 단계는 서비스별 과업을 3개에서 2개로 줄이면서, 남은 과업의 현실성을 높이고 과도한 조작 비용을 더 분명하게 드러나도록 정리한 작업이다. 중복되는 과업은 제거하고, 기존에 삭제했던 설정·옵션 선택류의 복잡성을 현재 3개 서비스 안에 필요한 만큼만 흡수했다.

연구 목적은 단순히 키보드 조작 가능 여부를 확인하는 것이 아니라, 목표 도달까지 발생하는 진입 비용, 반복 항목 비용, 복구 비용, 듣기 부담, 스위치 스캔 비용을 관찰하는 것이다. 따라서 이번 수정도 실제 서비스에서 자주 생길 법한 불필요한 초점 대상, 접힘 처리 실패, 장황한 조건 선택, 저장 옵션 선택 부담을 중심으로 구성했다.

## 2. 공통 변경

### 2.1 과업 수 축소

- 예약 캘린더: 3개 과업 → 2개 과업
- 댓글 목록: 3개 과업 → 2개 과업
- 검색 결과 목록: 3개 과업 → 2개 과업

`data/service-registry.js`의 `taskCount`도 서비스별 2개 기준으로 맞췄다. 홈 화면 진행 상태 계산은 저장된 과업 기록을 계속 보존하되, 새 기준의 전체 과업 수에 따라 `수행 전`, `진행 중`, `수행 완료`를 판단한다.

### 2.2 벤치마크 재생성

다음 파일을 2개 과업 기준으로 다시 구성했다.

그리고 `node scripts/run-benchmark.mjs`를 실행해 아래 결과 파일을 다시 생성했다.

## 3. 예약 캘린더 변경

### 3.1 과업 구성

남은 과업은 다음 2개다.

1. 김민아 상담사의 화요일 14:30 비대면 30분 시간으로 예약하고, 상담 주제는 직장 스트레스, 사전 작성은 간단히, 문자 알림은 받는 것으로 선택한다.
2. 기존 예약을 하나 취소한 뒤 박하늘 상담사의 목요일 10:00 대면 30분 시간으로 새로 예약한다.

삭제한 과업은 `현재 예약을 같은 날 더 이른 13:30 시간으로 변경` 과업이다.

### 3.2 전체 메뉴 접힘 처리

예약 캘린더 A/B 수행 화면의 글로벌 탐색 메뉴를 접고 펼칠 수 있는 형태로 바꿨다. 기본값은 접힘 상태다.

- 비교안 A: 접힌 메뉴가 시각적으로는 숨겨져 있지만, 메뉴 링크들이 여전히 Tab 이동 대상에 남아 있다.
- 비교안 B: 접힌 메뉴는 실제로 `hidden` 처리되어 Tab 이동 대상에서 빠진다.

이 차이는 숨김·접힘 상태의 항목이 초점 대상에 남아 있을 때 순차 탐색 비용이 늘어나는 문제를 보여 주기 위한 것이다.

### 3.3 상담 옵션 선택 추가

김민아 상담사의 화요일 14:30 비대면 30분 예약에는 예약 확정 전에 상담 옵션 화면이 열린다.

- 상담 주제: 관계 상담, 직장 스트레스, 수면과 생활 리듬, 가족 상담
- 사전 작성: 작성하지 않음, 간단히 작성, 자세히 작성
- 알림 방법: 이메일 알림, 문자 알림, 문자와 이메일 알림

성공 기준은 다음이다.

- 목표 예약 시간 선택
- 상담 주제 `직장 스트레스`
- 사전 작성 `간단히 작성`
- 알림 방법 `문자 알림`
- 예약 완료 화면 진입
- 이후 사용자가 직접 과업 종료

비교안 A의 상담 옵션은 시각적으로 접힌 선택값처럼 보이지만, 실제 초점 이동에서는 여러 선택지가 버튼처럼 반복 탐색되는 형태로 구성했다. 비교안 B는 일반적인 선택 컨트롤로 간결하게 구성했다.

### 3.4 한 주 최대 예약 2개 제한 추가

박하늘 상담사 목요일 10:00 대면 30분 예약 과업은 시작할 때 이미 예약 2개가 있는 상태로 시작한다.

- 김민아 상담사 화요일 14:30 비대면 30분
- 이서연 상담사 월요일 14:00 비대면 30분

한 주 최대 예약은 2개로 설정했다. 따라서 사용자가 기존 예약을 취소하지 않고 세 번째 예약을 시도하면, `한 주에 최대 2개까지 예약할 수 있습니다. 기존 예약을 먼저 취소해 주십시오.` 안내 대화상자가 열린다.

기존 예약 2개 중 어느 것을 취소해도 성공 후보로 인정한다. 최종 성공에는 박하늘 상담사 목요일 10:00 대면 30분 예약 완료가 필요하다.

## 4. 댓글 목록 변경

### 4.1 과업 구성

남은 과업은 다음 2개다.

1. 정렬 기준을 최신순으로 바꾸고, 후기 댓글만 보이게 한 뒤 민지 댓글의 답글 목록에서 두 번째 답글 작성자를 확인한다.
2. 도움이 많은 순으로 바꾸고, 운영자 안내 댓글의 댓글 정보 보기를 열었다가 닫은 뒤 도움이 돼요를 누른다.

삭제한 과업은 `질문 댓글만 보이게 바꾸고, 준호 댓글의 답글 목록에서 세 번째 답글 작성자 확인` 과업이다.

### 4.2 조건 선택 변형

비교안 A에서는 정렬 기준과 댓글 종류 선택이 화면상으로는 접힌 선택값처럼 보이지만, 실제 탐색에서는 여러 선택지가 버튼처럼 노출되는 구조로 바뀌었다. 접근 자체는 가능하지만, 조작 횟수와 발화 부담이 늘어나는 실제 서비스형 불편을 재현한다.

비교안 B는 일반 선택 컨트롤 중심으로 유지했다.

### 4.3 내부 판정 유지

댓글 과업은 수행 중 정답·오답을 알려 주지 않는다. 다만 내부 변수로 다음을 계속 기록한다.

- 정렬 기준을 과업 요구와 맞게 바꿨는지
- 댓글 종류를 과업 요구와 맞게 바꿨는지
- 지정 답글 목록에 실제로 진입했는지
- 현재 수행 탭에 표시된 무작위 작성자 이름 기준의 답을 제출했는지
- 댓글 정보 보기를 열고 닫았는지
- 목표 댓글에 도움이 돼요를 눌렀는지

## 5. 검색 결과 목록 변경

### 5.1 과업 구성

남은 과업은 다음 2개다.

1. 정렬 기준을 최신 자료 순으로 바꾸고, 안내문만 보이게 한 뒤 `예약 변경 안내`의 미리보기를 확인한다. 과업 종료 영역에서 예약 일시를 몇 시간 전까지 변경할 수 있는지 답한다.
2. 정렬 기준을 제목순으로 바꾸고, 질문답변만 보이게 한 뒤 `비대면 상담 연결 방법`을 개인 보관함에 저장한다. 저장할 때 상담 전 점검 항목을 포함한다.

삭제한 과업은 `신청 서식에서 상담 준비 체크 목록 미리보기 후 바로 열기` 과업이다.

### 5.2 단일 페이지 건너뛰기 링크 추가

비교안 B 최상단에 다음 링크를 추가했다.

- `검색 결과 영역으로 바로가기`
- `검색 조건 설정으로 바로가기`

이는 반복 블록이 페이지 여러 개에 걸쳐 있지 않더라도, 단일 화면 안의 긴 조건·결과 구조에서 주요 영역으로 건너뛰는 수단이 조작 비용을 줄일 수 있음을 보여 주기 위한 것이다.

### 5.3 미리보기 답변 과업

`예약 변경 안내` 미리보기에는 예약 일시 변경 기준이 포함된다. 사용자는 과업 종료 영역의 객관식 선택지에서 답을 고르고 `답변 제출하고 과업 종료하기`를 실행한다.

성공 기준은 다음이다.

- 정렬 기준 `최신 자료 순`
- 자료 범위 `안내문`
- `예약 변경 안내` 미리보기 실제 진입
- 현재 미리보기 내용 기준의 정답 선택
- 과업 종료

정답을 맞게 골라도 미리보기에 실제로 진입하지 않았다면 성공으로 기록하지 않는다.

### 5.4 저장 옵션 과업

`비대면 상담 연결 방법` 저장은 단순 저장 토스트에서 한 단계 더 실제 서비스에 가까운 저장 옵션 흐름으로 바꿨다.

- 저장 위치: 기본 보관함, 개인 보관함, 팀 공유함
- 포함할 내용: 요약만, 상담 전 점검 항목 포함, 전체 본문 포함
- 저장 형식: 웹 보관, PDF 사본, 링크만 저장

성공 기준은 다음이다.

- 정렬 기준 `제목순`
- 자료 범위 `질문답변`
- 목표 자료 `비대면 상담 연결 방법`
- 저장 위치 `개인 보관함`
- 포함할 내용 `상담 전 점검 항목 포함`
- 저장 형식 `웹 보관`
- 저장 완료

비교안 A의 저장 옵션은 버튼형 선택지가 반복되는 장황한 형태로 제공하고, 비교안 B는 선택 컨트롤로 간결하게 제공한다.

## 9. 남은 수동 점검

브라우저와 실제 화면낭독기 조합에서는 다음을 우선 확인한다.

- 예약 캘린더 A의 접힌 메뉴가 의도대로 초점 이동 대상에 남아 있는지
- 예약 캘린더 B의 접힌 메뉴가 초점 이동 대상에서 빠지는지
- 상담 옵션 선택 뒤 예약 확정 흐름이 자연스러운지
- 기존 예약 2개 상태에서 새 예약 시도 시 예약 제한 안내가 뜨는지
- 댓글 목록 A의 조건 선택 변형이 실제로 장황하지만 접근 가능한지
- 검색 결과 목록 B의 두 건너뛰기 링크가 올바른 영역으로 초점을 옮기는지
- 미리보기 답변과 저장 옵션 과업이 최종 종료 전까지 성공·실패 힌트를 주지 않는지
