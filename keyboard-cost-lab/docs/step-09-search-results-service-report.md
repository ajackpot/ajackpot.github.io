# 9단계 보고서

## 제목

검색 결과 목록 서비스 추가와 홈 연결·문서·사전 계산 결과 정리

## 이번 단계의 배경

브라우저 자동화는 끝단까지 완전하게 검증하기 어려우므로, 수동 점검표를 함께 배포하는 현재 운영 방식에 맞춰 다음 서비스 유형을 차례대로 확장하는 방향으로 작업을 이어 갔다. 이번 단계에서는 기존의 예약 캘린더, 댓글 목록, 상품 옵션 선택에 이어 네 번째 서비스 유형으로 `검색 결과 목록`을 추가했다.

검색 결과 목록은 다음과 같은 이유로 과도한 순차 탐색 부담을 잘 드러내는 유형이다.

- 상단 보조 링크와 검색 조건 선택을 지나야 실제 결과에 도달하는 구조가 흔하다.
- 결과마다 제목, 시각, 공유, 저장, 미리보기, 바로 열기 같은 조작이 흩어져 있으면 탭 이동 수가 길어진다.
- 미리보기 대화상자를 닫았을 때 원래 자료 작업으로 돌아오지 않으면 다시 위치를 찾아야 하는 부담이 생긴다.

## 이번 단계에서 추가한 서비스

### 1. 서비스 유형

- 서비스 이름: 검색 결과 목록
- 진입 파일: `search.html`
- 실행 스크립트: `search-app.js`

### 2. 과업 구성

이번 단계에서는 검색어 `예약 변경`을 기준으로 실험용 통합 검색 화면을 만들고, 아래 3개 과업을 추가했다.

1. 최신 안내문에서 `예약 변경 안내` 미리보기 열었다가 닫기
2. 제목순 질문답변에서 `비대면 상담 연결 방법` 저장
3. 신청 서식에서 `상담 준비 체크 목록` 미리보기 후 바로 열기

과업 내용은 모두 메인 창의 과업 준비 화면에서 먼저 읽고, 실제 조작은 새 탭에서만 수행하도록 기존 운영 방식을 그대로 따랐다.

## 비교안 구조

### 비교안 A

- 상단 보조 링크와 자료 조건 선택을 지난 뒤 검색 결과에 도달한다.
- 결과마다 제목 링크, 갱신 시각, 공유, 저장, 바로 열기, 미리보기가 각각 따로 있어 순차 이동이 길다.
- 미리보기 대화상자를 닫으면 원래 자료 작업이 아니라 검색 결과 제목 근처로 돌아와 다시 위치를 찾아야 한다.

### 비교안 B

- 맨 앞의 `검색 결과로 바로 이동` 링크로 첫 진입 부담을 줄인다.
- 결과는 하나의 선택 항목으로 제공하고 방향키로 이동한다.
- `선택한 자료 작업` 영역에서 미리보기, 저장, 바로 열기를 이어서 수행한다.
- 미리보기 대화상자를 닫으면 방금 사용한 작업 버튼으로 초점이 돌아온다.

## 홈 화면과 공통 구조 반영

- 홈 화면 서비스 카드에 `검색 결과 목록`을 추가했다.
- 서비스 카드 버튼은 기존 원칙대로 `검색 결과 목록 화면으로 이동`처럼 대상 이름을 포함한 문구를 유지했다.
- 홈에서 검색 결과 목록을 선택하면 `search.html`로 이동하도록 연결했다.
- 비교안 순서는 검색 결과 목록도 다른 서비스와 같이 `A → B`로 고정했다.
- 과업 성공 뒤 완료 대화상자를 띄우고, `확인`을 누르면 수행 탭이 닫히는 공통 흐름을 그대로 사용했다.

## 화면 문구와 용어 정리

이번 단계에서도 SC 3.1.3 대응 원칙에 맞춰 낯선 표현을 설명 가능한 한국어 중심 문구로 정리했다.

추가 반영한 주요 용어는 다음과 같다.

- `search result` → `검색 결과`
- `preview` → `미리보기`
- `save` → `저장`
- `relevance` → `정확도순`

또한 용어 가이드의 서비스 화면 규칙을 현재 UI에 맞춰 `과업 준비로 이동` 표현으로 정리했다.

## 사전 계산 기준 결과

`node scripts/run-benchmark.mjs`를 다시 실행해 검색 결과 목록용 사전 계산 결과를 생성했다.

기준 예상 합계는 다음과 같다.

- 키보드 사용자: A 108.7초 → B 41.1초, 67.6초 감소
- 화면낭독 사용자: A 189.3초 → B 68.8초, 120.5초 감소
- 스위치 사용자: A 265.5초 → B 95.1초, 170.4초 감소

즉, 검색 결과 목록에서도 비교안 B가 세 사용자 유형 모두에서 큰 폭의 예상 감소를 보였다.

## 수동 점검표 반영

브라우저 자동화 대신 사람이 확인할 수 있도록 수동 점검표에도 검색 결과 목록 항목을 추가했다.

추가한 확인 항목은 아래와 같다.

- 홈에서 검색 결과 목록 서비스로 진입 가능한지
- 과업 준비 화면에서 현재 검색 조건과 목표 자료가 보이는지
- 비교안 A/B 순서가 고정되어 있는지
- 수행 탭 하단에 `검색 결과로 이동`, `이 탭 닫기` 두 버튼만 보이는지
- 비교안 A에서는 결과마다 분산된 작업 구조와 미리보기 닫기 뒤 위치 다시 찾기가 나타나는지
- 비교안 B에서는 하나의 선택 항목 구조와 작업 묶음, 미리보기 닫기 뒤 초점 복귀가 나타나는지

## 남은 수동 점검 권장 사항

자동화 대신 실제 브라우저에서 아래 항목을 한 번 더 확인하면 좋다.

- 검색 결과 목록 서비스 카드에서 새 화면으로 진입하는지
- 메인 창에서 과업을 확인한 뒤 새 탭에서 과업 수행이 시작되는지
- 비교안 A에서 미리보기를 닫으면 검색 결과 제목 근처로 돌아오는지
- 비교안 B에서 미리보기를 닫으면 방금 사용한 작업 버튼으로 돌아오는지
- 과업 성공 뒤 완료 대화상자가 뜨고 `확인`으로 새 탭이 닫히는지
