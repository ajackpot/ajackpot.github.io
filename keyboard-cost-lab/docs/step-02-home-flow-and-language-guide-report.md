# 2단계 보고서 · 서비스 선택 흐름 추가 + 화면 문구 정비

작성 시각: 현재 작업 결과 기준

## 1. 이번 단계 목표

이번 단계의 목표는 두 가지였다.

1. 홈 화면에서 바로 과업을 시작하지 않고, 먼저 서비스 유형을 선택한 뒤 해당 서비스 화면에서 과업을 시작하도록 흐름을 바꾼다.
2. 모든 화면에서 낯선 영어 표현을 줄이고, WCAG 2.2 SC 3.1.3과 한국형 웹 접근성 관련 용어를 반영한 문구 기준을 만든다.

## 2. 완료한 변경 사항

### 2.1 화면 흐름 변경

기존 흐름:

- 첫 화면 → 바로 실험 시작

변경 후 흐름:

- 홈 화면 → 서비스 유형 선택 → 서비스 화면 → 과업 시작

적용 내용:

- 홈 화면에 서비스 유형 카드 추가
- 예약 캘린더만 현재 사용 가능 상태로 표시
- 예약 캘린더 서비스 화면에서 `과업 시작` 또는 `서비스 선택으로 돌아가기` 가능
- `처음부터 다시 시작`은 홈이 아니라 현재 서비스 화면으로 돌아가도록 조정

### 2.2 문구 정비

주요 변경 예시는 다음과 같다.

| 이전 표현 | 변경 후 표현 |
|---|---|
| 세션 ID | 실험 번호 |
| 필터 | 조건 선택 |
| 필터 적용 | 조건 적용 |
| 슬롯 | 예약 시간 |
| 결과 영역으로 건너뛰기 | 결과로 바로 이동 |
| 모달 | 대화상자 |
| 포커스 | 초점 |
| backtrack 입력 | 되돌아간 입력 |
| wrong selection | 목표와 다른 시간 선택 |
| modal escape | 대화상자 바깥으로 초점 이탈 |
| pointer activation | 클릭 입력 |
| 벤치마크 | 사전 계산 기준 |
| 프로필 | 사용자 유형 |

### 2.3 용어 설명 제공

모든 화면 하단에 `용어 설명 보기`를 추가했다.

현재 포함 용어:

- 비교안 A/B
- 사전 계산 기준
- 초점
- 대화상자
- 결과 파일(JSON)

### 2.4 사전 계산 기준 문구도 동기화

사전 계산 그래프 원본의 설명 문구를 화면 문구 기준에 맞게 수정한 뒤 결과 파일을 다시 생성했다.

수정 예시:

- `헤더/필터 이후 결과 진입, 슬롯별 복수 탭 스톱, 약한 모달 포커스 관리.`
  → `상단 링크와 조건 선택을 지난 뒤 결과에 도달하고, 예약 시간마다 여러 번 멈춰야 하며, 대화상자 초점 관리가 약한 구조.`

- `스킵 링크, 결과 이동, 슬롯 그리드 단일 진입, 모달 초기 포커스·복귀 보장.`
  → `결과로 바로 이동하고, 예약 시간표에 한 번만 들어간 뒤 이동하며, 대화상자 초점 이동과 복귀를 보장하는 구조.`

## 4. 검증한 내용

확인 완료:

- `node --check app.js` 문법 검사 통과
- `node scripts/run-benchmark.mjs` 실행 완료
- 사전 계산 결과 파일 재생성 완료

이번 단계에서 아직 직접 끝까지 확인하지 못한 것:

- 브라우저에서 실제 키보드로 처음부터 끝까지 수행하는 수동 점검
- 화면낭독기 실사용 점검

즉, 정적 코드와 계산 결과는 갱신되었고, 상호작용 흐름의 최종 사용성 확인은 다음 단계에서 한 번 더 점검하는 것이 좋다.

## 5. 다음 단계에 바로 이어질 항목

1. 예약 캘린더 화면을 실제 키보드로 처음부터 끝까지 점검하며 잔여 문구와 초점 흐름을 교정
2. 설문지 주소를 실제 값으로 연결하고 전달 항목 확정
3. 다음 서비스 유형 하나를 골라 같은 틀로 추가 구현

## 6. 이번 단계에서 유지한 원칙

- 서비스 선택은 과업 시작 전의 고정 단계로 두고, A/B 비교 대상에서 분리한다.
- 낯선 영어 표현보다 이해하기 쉬운 한국어를 우선한다.
- 꼭 남겨야 하는 용어는 설명을 함께 제공한다.
- 단계가 끝날 때마다 `docs` 폴더에 보고서를 남긴다.
