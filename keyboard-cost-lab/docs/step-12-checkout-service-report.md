# 12단계 보고서

부제: 신청·결제 흐름 서비스 추가와 홈 서비스 경로 정리

## 1. 이번 단계 목표

이번 단계에서는 기존 실험 틀을 유지한 채 새 서비스 유형 `신청·결제 흐름`을 추가했다. 홈 화면에서 서비스 카드로 진입하고, 메인 창에서 과업을 확인한 뒤, 새 탭에서 비교안 A/B를 수행하고, 사전 계산 기준과 실제 기록을 비교하는 흐름을 그대로 재사용했다.

함께 한 작은 정리는 다음과 같다.

- 홈 화면의 서비스 정보 묶음에 서비스별 경로를 함께 두어 새 서비스를 추가할 때 라우팅 수정 범위를 줄였다.
- 신청·결제 흐름은 기존 설정 화면 계열의 묶음형 구조를 바탕으로, `신청 정보 → 안내 수신 → 결제 수단 → 제출 확인` 단계로 재구성했다.
- 제출 동작은 단계별 저장이 아니라 최종 `신청서 제출`로 정리해 서비스 맥락을 더 자연스럽게 맞췄다.

## 2. 구현 범위

### 2.1 홈 화면

- `신청·결제 흐름` 서비스 카드를 추가했다.
- 서비스 카드 정보에 `path`를 둘 수 있게 정리해 `comments.html`, `product.html`, `search.html`, `settings.html`, `filters.html`, `checkout.html`로 공통 처리하도록 바꿨다.
- 홈 안내 문단은 현재 공개된 서비스 이름을 서비스 정보 묶음에서 직접 읽어 오도록 바꿨다.

### 2.2 새 서비스 파일

아래 파일을 새로 추가했다.

- `checkout.html`
- `checkout-app.js`
- `data/checkout-scenario.js`
- `data/tasks-checkout.js`
- `data/benchmark-graphs-checkout.js`
- `data/benchmark-results-checkout.json`
- `data/benchmark-results-checkout.js`

### 2.3 신청·결제 흐름 A/B 구조

#### 비교안 A

- 상단 보조 링크와 여러 신청 단계 안내를 지나야 원하는 항목에 도달한다.
- 각 항목마다 `현재 선택`, `최근 확인`, `설명 보기`, `값 선택`이 따로 있어 순차 이동이 길다.
- 설명 대화상자를 닫으면 방금 사용한 항목으로 돌아가지 않고 `신청 단계` 제목 근처로 돌아온다.
- `신청서 제출`은 마지막 `제출 확인` 단계에 있다.

#### 비교안 B

- `신청 단계로 바로 이동` 링크로 첫 진입 부담을 줄인다.
- 단계 목록은 한 번만 들어간 뒤 방향키로 이동한다.
- 같은 단계 안에서 설명 보기와 값 변경을 이어서 수행할 수 있다.
- 설명 대화상자를 닫으면 방금 사용한 설명 보기 버튼으로 초점이 돌아온다.
- `제출 확인` 단계에서 `신청서 제출`로 과업을 마무리한다.

## 3. 과업 구성

### 과업 1

`비대면 초진 상담을 고르고 연락처 확인 안내를 켠 뒤 카드 결제로 신청 완료`

- 상담 종류를 `비대면 초진 상담`으로 변경
- 연락처 확인 안내 `켜기`
- 결제 수단 `카드 결제`
- 제출 확인에서 `신청서 제출`

### 과업 2

`보호자 대리 신청과 준비 안내 설명 확인 뒤 계좌 이체로 신청 완료`

- 신청 대상을 `보호자 대리 신청`으로 변경
- `준비 안내 받는 방법` 설명 열기/닫기
- 준비 안내 받는 방법 `문자`
- 결제 수단 `계좌 이체`
- 제출 확인에서 `신청서 제출`

### 과업 3

`재방문 빠른 상담과 문자 영수증, 간편 결제로 신청 완료`

- 상담 종류를 `재방문 빠른 상담`으로 변경
- 영수증 받는 방법 `문자`
- 결제 수단 `간편 결제`
- 제출 확인에서 `신청서 제출`

## 8. 다음 단계 메모

신청·결제 흐름까지 추가되면서 서비스 유형이 더 늘어났으므로, 다음 단계에서는 다음 중 하나를 선택해 진행하기 좋다.

1. 이번까지 만든 일곱 서비스의 공통 코드를 더 묶는 리팩터링
2. 신청 완료 뒤 영수증 확인, 결제 취소, 신청 내역 수정 같은 후속 하위 흐름 추가
3. 새 서비스 유형 추가
