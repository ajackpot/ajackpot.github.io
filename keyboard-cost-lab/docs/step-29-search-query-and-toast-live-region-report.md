# Step 29 검색 조건 및 토스트 알림 개선 보고서

## 반영 범위

- 기준 프로젝트: `keyboard-cost-lab-step-27-language-cleanup-full.zip`
- 수정 서비스: 검색 결과 목록, 전체 서비스 공통 토스트 알림

## 검색 결과 목록 변경

1. 상단 헤더의 기존 자료 검색 폼을 제거했습니다.
2. 기존 `정렬과 자료 범위 선택` 영역 제목을 `검색 조건 선택`으로 바꿨습니다.
3. `검색어` 편집창을 `자료 범위` 다음 항목으로 이동했습니다.
4. `조건 적용`을 누르거나 검색어 편집창에서 Enter를 누르면 다음 조건이 함께 적용됩니다.
   - 정렬 기준
   - 자료 범위
   - 검색어
5. 검색어는 자료 제목, 자료 유형, 요약, 본문, 관련 낱말을 기준으로 걸러집니다.
6. 검색어가 없으면 기존처럼 자료 범위와 정렬 기준만 적용됩니다.

## 토스트 알림 개선

기존에는 하단 토스트 요소가 메시지와 함께 새로 만들어졌습니다. 이 방식은 화면 낭독기가 live region 생성을 놓칠 수 있습니다.

이번 수정에서는 다음 구조로 바꿨습니다.

1. 화면에 보이는 하단 토스트는 기존 위치와 모양을 유지합니다.
2. 화면 낭독기 알림용 숨김 영역을 `body` 아래에 한 번 만들고 계속 유지합니다.
3. 토스트 메시지가 바뀔 때마다 숨김 live region의 내용을 비운 뒤 다시 넣어 새 상태 메시지로 읽히도록 했습니다.
4. 이 방식은 예약 캘린더, 댓글 목록, 검색 결과 목록의 과업 수행 페이지에 공통 적용했습니다.

## 검증

다음 검사를 통과했습니다.

```bash
node --check lib/utils.js
node --check app.js
node --check comments-app.js
node --check search-app.js
npm run check
```

`npm run check`에는 정적 품질 검사, 정적 UI smoke test, 계약 검사, 댓글 작성자 무작위 배정 검사, 수행 화면 렌더링 smoke test가 포함됩니다.
