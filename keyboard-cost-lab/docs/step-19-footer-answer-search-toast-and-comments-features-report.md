# 19단계 보고서: 댓글 답변 종료 영역 통합, 검색 결과 토스트화, 댓글 보조 기능 강화

## 배경

18단계 이후 댓글 목록 과업에서는 답글 목록에 진입한 뒤 `답변 제출` 버튼을 누르고, 다시 하단의 `과업 종료` 버튼을 눌러야 했다. 이 구조는 실제 댓글 탐색 과업보다 실험 검증용 동작이 더해져 조작 횟수와 완료 시간이 늘어나는 문제가 있었다. 또한 검색 결과 목록은 저장, 바로 열기, 미리보기 닫기 뒤에 일반적인 처리 완료 대화상자를 띄워 실제 서비스보다 실험용 화면처럼 보였다. 댓글 목록의 상단 메뉴와 보조 링크도 점검 중 안내가 많아 현실감이 부족했다.

## 변경 요약

이번 단계에서는 다음을 반영했다.

1. 댓글 답글 질문의 답변 제출을 수행 탭 하단의 과업 종료 영역으로 옮겼다.
2. 답변이 필요한 과업의 하단 버튼 이름을 `답변 제출하고 과업 종료하기`로 바꿨다.
3. 과업 종료 확인 대화상자의 확인 버튼은 답변 과업에서 `예, 제출하고 종료합니다`로 표시한다.
4. 답글 목록 안의 별도 `답변 제출` 버튼과 질문 양식은 제거했다.
5. 검색 결과 목록의 저장과 바로 열기는 처리 완료 대화상자 대신 토스트 알림으로 처리한다.
6. 검색 결과 미리보기 대화상자는 닫으면 이전 흐름으로 돌아가며, 별도 안내 메시지를 띄우지 않는다.
7. 검색 결과의 최종 판정은 더 이상 처리 완료 대화상자 확인 여부에 의존하지 않는다.
8. 댓글 목록 상단 메뉴, 게시글 보관, 신고, 이용 안내, 내 댓글, 작성 안내, 작성자 정보, 댓글 시각, 공유 등을 화면 안 패널·토스트·상태 변경으로 보강했다.

## 댓글 답변 과업 변경

기존 구조는 다음과 같았다.

- 목표 댓글의 답글 목록 열기
- 답글 목록 안에서 객관식 답변 선택
- `답변 제출` 실행
- 하단 `과업 종료` 실행
- 종료 확인 대화상자에서 `예` 실행

새 구조는 다음과 같다.

- 목표 댓글의 답글 목록 열기
- 하단 과업 종료 영역에서 답변 선택
- `답변 제출하고 과업 종료하기` 실행
- 종료 확인 대화상자에서 `예, 제출하고 종료합니다` 실행

답변 선택은 실험 검증용 입력에 가깝기 때문에 하단 운영 영역 안에 두었다. 따라서 댓글 서비스 본문 안의 탐색 비용을 불필요하게 늘리지 않는다. 내부 판정은 여전히 다음을 모두 본다.

- 요청한 댓글의 답글 목록에 실제로 진입했는가?
- 하단 답변 선택에서 값을 골랐는가?
- 고른 값이 요청한 답글 정보와 맞는가?
- 다른 댓글의 관련 동작을 먼저 수행하지 않았는가?

즉, 답을 우연히 맞게 고르더라도 지정된 답글 목록에 진입하지 않았다면 성공으로 기록하지 않는다.

## 검색 결과 목록 피드백 변경

검색 결과 목록의 `저장`과 `바로 열기`는 이제 다음처럼 처리한다.

- `저장`: 화면 하단 토스트 `자료를 저장했습니다.` 또는 `이미 저장한 자료입니다.` 표시
- `바로 열기`: 화면 하단 토스트 `자료를 열었습니다.` 표시와 자료 상태 카드 표시
- `미리보기 닫기`: 별도 메시지 없이 이전 흐름으로 복귀

수행 중 성공·실패 힌트는 계속 표시하지 않는다. 토스트는 실제 서비스 피드백처럼 상태만 알려 주며, 그 동작이 과업 성공 조건에 맞는지는 과업 종료 뒤 최종 결과에서만 확인한다.

## 댓글 목록 보조 기능 강화

다음 기능은 더 이상 단순 점검 중 안내만 띄우지 않는다.

- 마음돌봄 커뮤니티 처음 화면
- 게시글 목록
- 인기 글
- 이용 안내
- 새 글 쓰기 양식
- 알림과 알림 설정
- 내 댓글과 보관한 글 요약
- 커뮤니티 규칙
- 작성 안내
- 문의 접수
- 게시글 신고
- 작성자 정보
- 댓글 작성 시각 정보
- 게시글 보관
- 공유 주소 복사

복잡한 서버 저장이나 실제 페이지 이동은 하지 않지만, 수행 탭 안에서 패널, 토스트, 임시 저장, 보관 상태처럼 실제 서비스에 가까운 반응을 제공한다.

## 사전 계산 결과 갱신

댓글 답변 제출 위치가 바뀌면서 댓글 목록 벤치마크 그래프도 갱신했다. 답변 선택은 하단 운영 영역에서 처리하므로 댓글 본문 탐색 비용에서 제외했다.

새 댓글 목록 기준 예상 합계는 다음과 같다.

- 키보드 사용자: A 101.0초 → B 34.1초
- 화면낭독 사용자: A 174.4초 → B 56.7초
- 스위치 사용자: A 247.9초 → B 78.0초

## 남은 수동 점검

브라우저와 화면낭독기 조합에서는 다음을 우선 확인한다.

- 댓글 답글 질문 과업에서 하단 답변 선택과 `답변 제출하고 과업 종료하기`가 자연스럽게 탐색되는지
- 답글 목록에 들어가지 않고 답변만 고른 경우 성공으로 기록되지 않는지
- 검색 결과 저장과 바로 열기에서 대화상자가 아니라 토스트가 표시되는지
- 검색 결과 미리보기 닫기에서 별도 안내 메시지가 나오지 않는지
- 댓글 보조 메뉴가 점검 중 안내만 반복하지 않고 화면 안 패널로 열리는지
- 보조 기능 사용 중에도 과업 성공·실패 힌트가 표시되지 않는지
