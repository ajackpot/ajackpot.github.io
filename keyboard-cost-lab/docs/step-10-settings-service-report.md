# 10단계 보고서

부제: 설정 화면 서비스 유형 추가와 사전 계산 기준 확장

## 1. 이번 단계 목표

이번 단계에서는 `설정 화면` 서비스 유형을 기존 실험 틀에 맞춰 추가했다. 목표는 다음과 같았다.

- 홈 화면에서 `설정 화면`을 구현 완료 상태로 선택할 수 있게 한다.
- 메인 창에서 과업을 확인하고, 새 탭에서 비교안 A/B를 수행하는 기존 흐름을 그대로 유지한다.
- 설명 보기 대화상자, 설정 값 변경, 설정 묶음 저장을 포함한 과업 3개를 만든다.
- 설정 화면용 사전 계산 그래프와 결과 파일을 추가한다.
- 용어 가이드, 수동 점검표, README를 새 서비스 기준으로 갱신한다.

## 2. 구현한 내용

### 2.1 새 서비스 유형 추가

홈 화면에서 `설정 화면` 카드를 선택할 수 있도록 `app.js`의 서비스 목록과 이동 분기를 갱신했다.

- 서비스 이름: `설정 화면`
- 서비스 요약: 설명 보기, 설정 값 변경, 설정 묶음 저장 과업 비교
- 진입 파일: `settings.html`
- 실행 파일: `settings-app.js`

### 2.2 설정 화면 시나리오와 과업

새 데이터 파일을 추가했다.

- `data/settings-scenario.js`
- `data/tasks-settings.js`
- `data/benchmark-graphs-settings.js`

설정 묶음은 다음 네 가지로 구성했다.

- 알림 설정
- 예약 편의
- 보안 설정
- 화면 설정

과업은 다음 세 가지다.

1. 알림 설정에서 문자 알림 끄고 상담 하루 전 알림 켜기
2. 보안 설정에서 로그인 확인 단계 설명을 열었다가 닫고 새 기기 로그인 알림 켜기
3. 화면 설정에서 글자 크기 110%와 높은 대비 켜기

### 2.3 비교안 A/B 구조

비교안 A는 길게 이어진 설정 묶음을 순차적으로 지나야 하고, 각 설정마다 설명 보기와 값 선택 버튼이 흩어져 있는 구조로 만들었다.

- 상단 보조 링크 뒤에 긴 설정 목록 배치
- 각 설정에 현재 상태, 최근 변경, 설명 보기, 값 선택 버튼 분산
- 설명 대화상자를 닫으면 설정 화면 제목 근처로 복귀
- 저장 버튼은 각 설정 묶음 아래에 별도 배치

비교안 B는 빠른 이동과 묶음 단위 선택을 통해 반복 이동 비용을 줄이는 구조로 만들었다.

- `설정 항목으로 바로 이동` 링크 제공
- 설정 묶음을 한 번만 들어간 뒤 방향키로 선택하는 구조
- 같은 묶음 안에서 설명 보기, 값 변경, 저장을 가깝게 배치
- 설명 대화상자를 닫으면 같은 설명 보기 버튼으로 초점 복귀

### 2.4 계측과 완료 처리

기존 서비스들과 같은 운영 원칙을 그대로 유지했다.

- 실제 계측은 수행 탭 첫 조작부터 시작
- 숨김 탭 시간 제외
- 수행 탭 하단 보조 버튼 영역은 측정 제외
- 과업 성공 뒤 완료 안내 팝업 표시
- `확인` 버튼으로 수행 탭 자동 닫기

- 스위치 사용자: A 221.4초 → B 71.5초
