# 17단계 보고서: 예약 캘린더 보조 기능 현실감 강화

## 1. 작업 배경

이번 단계는 예약 캘린더 수행 탭이 너무 가짜 페이지처럼 보이는 문제를 줄이기 위한 정비다. 실험의 핵심은 목표 도달까지의 조작 비용을 보는 것이므로, 과업 성공 여부를 암시하는 힌트는 계속 금지한다. 대신 실제 서비스에 있을 법한 검색, 알림, 내 상담, 상담사 소개, 이용권, 가격 안내 같은 보조 기능을 화면 안에서 자연스럽게 반응하도록 만들었다.

## 2. 구현 원칙

- 예약 확정, 예약 취소, 과업 종료, 최종 결과 판정 흐름은 변경하지 않는다.
- 과업 수행 중 성공·실패·정답 여부를 암시하는 메시지는 추가하지 않는다.
- 구현이 간단하거나 설명 패널로 충분한 기능은 `현재 점검 중` 대신 실제 기능처럼 반응하게 한다.
- 구현 범위가 큰 기능은 여전히 점검 중 안내를 유지하되, 페이지 밖으로 이동하지 않는다.
- 보조 기능은 실제 서비스 콘텐츠로 취급하므로 수행 중 사용자가 접근하면 일반 탐색 비용에 포함될 수 있다.

## 3. 변경 내용

### 3.1 예약 캘린더 상단 기능 강화

기존에는 상단 메뉴 대부분이 `현재 점검 중` 안내만 출력했다. 이번 단계에서는 다음 기능을 화면 안 패널로 구현했다.

- `처음 화면`: 최근 공지와 빠른 메뉴 표시
- `상담사 소개`: 상담사별 분야, 다음 예약 가능 시간, 상담사 일정 조건 입력
- `이용권`: 1회 상담권, 3회 묶음권, 청년 할인권 선택 상태 표시
- `이용 후기`: 최근 후기와 보관 상태 변경
- `가격 안내`: 30분/45분 상담 비용 표
- `자주 묻는 질문`: 비대면 상담, 예약 변경, 상담 전 메모 안내
- `운영 정책` 및 `변경 기준 보기`: 변경 가능 시간, 취소와 환불, 개인정보 보관 안내
- `문의`: 문의 접수 양식과 고객센터 안내
- `알림`: 최근 알림과 이메일/문자 알림 설정 전환
- `내 상담`: 현재 예약 내용, 최근 이용 내역, 상담 전 안내, 결제 내역
- `검색`: 검색 결과 패널과 상담 방식 조건 입력

### 3.2 여전히 점검 중으로 남긴 기능

`실시간 채팅 열기`는 실제 상담 연결 흐름과 별도 채팅 UI가 필요하므로 이번 단계에서는 점검 중 안내를 유지했다. 다만 페이지 밖으로 이동하지 않고 같은 수행 탭 안에서 안내만 표시한다.

### 3.3 상태 보존

수행 탭 내부 상태에 다음 값을 추가했다.

- `featurePanel`: 현재 열린 보조 기능 패널
- `savedFeatureItems`: 보관한 상담사 소개나 후기
- `selectedPass`: 선택한 이용권
- `reminderSettings`: 이메일/문자 알림 설정

이 값들은 수행 탭과 메인 창 간 결과 전달을 위해 런타임 스냅샷에도 포함된다.

### 3.4 정적 점검 강화

`scripts/smoke-render-runner.mjs`를 보강해 예약 캘린더 보조 기능 패널도 렌더링 점검 대상에 포함했다. 다음 패널들이 오류 없이 렌더링되는지 확인한다.

- home
- providers
- passes
- reviews
- pricing
- faq
- policy
- support
- notifications
- my-counseling
- usage-guide
- search

## 4. 수정 파일

- `app.js`
  - 예약 캘린더 보조 기능 패널 렌더링 추가
  - 상단 메뉴와 검색, 알림, 내 상담의 점검 중 안내를 실제 패널 동작으로 변경
  - 상담사 일정 조건 입력, 상담 방식 조건 입력, 이용권 선택, 보관, 알림 설정, 문의 접수 동작 추가
  - 보조 기능 상태 변경 뒤 초점이 가능한 한 원래 조작 위치에 남도록 보정
  - 런타임 스냅샷에 보조 기능 상태 추가
- `styles.css`
  - 보조 기능 패널, 작은 카드, 기능 목록 스타일 추가
- `scripts/smoke-render-runner.mjs`
  - 예약 캘린더 보조 기능 패널 렌더링 점검 추가
- `README.md`
  - 예약 캘린더 보조 기능 현실감 강화 내용 반영
- `docs/manual-release-checklist.md`
  - 예약 캘린더 보조 기능 수동 점검 항목 추가
- `docs/language-guideline-wcag-3.1.3-kwcag.md`
  - 점검 중 안내 사용 기준을 “간단 기능은 실제 반응, 복잡 기능만 점검 중”으로 조정

## 5. 검증

실행한 검증 명령은 다음과 같다.

```bash
node --check app.js
node --check scripts/smoke-render-runner.mjs
node scripts/run-benchmark.mjs
npm run check
```

`npm run check`에서 다음을 확인했다.

- 예약 캘린더 A/B 수행 화면 렌더링
- 예약 캘린더 B 시간표 버튼의 일반 실행 경로
- 내부 식별값 비노출
- 과업 요청 사항 수행 탭 표시 옵션
- 과업 종료 확인 대화상자에서 결과 메시지 미노출
- 예약 캘린더 보조 기능 패널 12개 렌더링
- 댓글 목록과 검색 결과 목록 기존 렌더링 유지

## 6. 남은 수동 점검

- 실제 브라우저에서 상단 메뉴 8개와 알림, 내 상담, 검색을 키보드로 열고 닫을 수 있는지 확인한다.
- 상담사 일정 조건 입력과 상담 방식 조건 입력 후 조건 적용 버튼으로 초점이 이동하는지 확인한다.
- 보조 기능 탐색 중에도 과업 성공·실패 힌트가 나오지 않는지 확인한다.
- 예약 확정, 최종 확인, 과업 종료 확인, 기록 저장 대화상자의 기존 흐름이 유지되는지 확인한다.
