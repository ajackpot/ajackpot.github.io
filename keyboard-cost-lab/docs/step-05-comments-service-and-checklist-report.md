# 5단계 보고서

제목: 댓글 목록 서비스 추가와 배포용 수동 점검표 정리

## 1. 이번 단계 목표

- 브라우저 자동화가 어려운 항목을 수동 점검표로 정리한다.
- 다음 서비스 유형으로 `댓글 목록` A/B 페이지를 추가한다.
- 홈 화면에서 예약 캘린더와 댓글 목록을 모두 선택할 수 있게 한다.
- 댓글 목록 서비스도 기존과 같은 흐름으로 운영한다.
  - 메인 창에서 과업 안내 확인
  - 새 탭에서 실제 수행
  - 첫 조작부터 계측 시작
  - 숨김 탭 시간 제외

## 2. 구현 완료 범위

### 2.1 홈 화면 확장

- `댓글 목록` 카드를 `구현 완료` 상태로 올렸다.
- 홈 화면에서 댓글 목록 카드를 누르면 `comments.html`로 이동하도록 연결했다.
- 현재 공개 범위 표시에 준비된 서비스 유형이 함께 나타나도록 바꿨다.

### 2.2 댓글 목록 서비스 추가

새 파일:

- `comments.html`
- `comments-app.js`
- `data/comments-scenario.js`
- `data/tasks-comments.js`
- `data/benchmark-graphs-comments.js`
- `data/benchmark-results-comments.json`
- `data/benchmark-results-comments.js`

설계 방향:

- 비교안 A: 댓글마다 여러 링크와 버튼이 흩어져 있어 순차 탐색 부담이 큰 구조
- 비교안 B: 댓글을 하나의 선택 항목으로 고르고, 댓글 작업을 한곳에 모은 구조
- 댓글 정보 대화상자 닫기 후 초점 복귀 차이까지 비교

### 2.3 댓글 목록 과업 3종 추가

1. 최신 후기에서 민지 댓글 답글 열기
2. 운영자 안내 댓글 정보 보기 후 도움이 돼요
3. 질문 댓글에서 준호 댓글 답글 열기

모든 과업은 메인 창에서 목표와 현재 상태를 먼저 보여 준 뒤, 새 탭에서 실제 수행하도록 만들었다.

### 2.4 수동 점검표 문서 추가

새 문서:

- `docs/manual-release-checklist.md`

포함 내용:

- 공통 흐름 점검
- 계측 규칙 점검
- 언어와 이해 가능성 점검
- 예약 캘린더 서비스 점검
- 댓글 목록 서비스 점검
- 결과 저장 및 문서 반영 점검

## 3. 벤치마크 구조 정리

기존 스크립트 `scripts/run-benchmark.mjs`를 확장해 다음 두 서비스를 함께 생성하도록 바꿨다.

- 예약 캘린더
- 댓글 목록

댓글 목록도 예약 캘린더와 같은 형식의 결과 파일을 생성하므로, 서비스별 비교 문서와 실제 기록 화면을 같은 방식으로 유지할 수 있다.

## 4. 언어와 용어 정비

기존 용어 가이드에 댓글 목록 서비스에서 자주 쓰이는 항목을 보강했다.

- `reply` → `답글`
- `sort` → `정렬 기준`

화면에는 내부 구현 용어 대신 아래 표현을 사용했다.

- `댓글 범위`
- `댓글 정보 보기`
- `선택한 댓글 작업`
- `도움이 돼요`

## 5. 검증 내용

실행한 검증:

- `node --check app.js`
- `node --check comments-app.js`
- `node --check scripts/run-benchmark.mjs`
- `node scripts/run-benchmark.mjs`

확인한 결과:

- 댓글 목록 사전 계산 결과 파일이 정상 생성됨
- 예약 캘린더 결과 파일도 함께 다시 생성됨
- 홈 앱과 댓글 목록 앱의 문법 오류 없음

## 6. 남은 메모

- 현재는 서비스별로 독립된 실험 페이지를 유지한다.
  - 홈 화면: `index.html`
  - 댓글 목록: `comments.html`
- 이 구조는 다음 서비스 유형을 추가할 때도 같은 방식으로 확장하기 쉽다.
- 브라우저 자동화가 어려운 부분은 `docs/manual-release-checklist.md`로 보완한다.
