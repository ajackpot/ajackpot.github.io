# 15단계 보고서: 과업 중 힌트 제거, 종료 확인 대화상자, 예약 캘린더 B 화면 오류 수정

## 1. 작업 배경

이 단계에서는 수행 탭 안에서 과업 성공 여부나 오답 여부를 알려 주는 메시지가 보이는 문제를 먼저 정리했다. 실험 화면은 사용자가 과업을 해결하는 공간이므로, 수행 중에 `성공`, `목표와 다름`, `아직 완료되지 않음` 같은 판정 정보를 제공하면 실제 탐색·판단 부담을 왜곡할 수 있다.

또한 `과업 종료` 버튼을 실수로 누르면 바로 기록이 저장될 수 있는 문제가 있어, 종료 확인 대화상자를 추가했다. 예약 캘린더의 두 번째 A/B 화면에서 콘텐츠가 탐색되지 않는다는 보고도 함께 확인했다.

## 2. 주요 변경 사항

### 2.1 과업 수행 중 힌트 제거

- 예약 캘린더, 댓글 목록, 검색 결과 목록 수행 탭에서 숨은 상태 영역 `live-status-region` 출력을 제거했다.
- 수행 중 `성공 여부`, `목표와 다른 항목 선택`, `완료 조건 재확인`을 알려 주던 안내 문구를 사용자 화면에 노출하지 않도록 정리했다.
- 비교안 A/B의 `방향키로 이동`, `탭 키로 차례대로 이동` 같은 키 조작 안내 문구도 수행 탭 콘텐츠에서 제거했다.
- 실제 사이트 보조 기능의 `현재 점검 중` 안내는 유지했다. 이는 과업 성공 힌트가 아니라 실행되지 않는 기능의 상태 안내이기 때문이다.

### 2.2 서비스 확인 대화상자 문구 일반화

- 주요 처리 동작 뒤 뜨는 서비스 확인 대화상자를 `처리가 완료되었습니다.` 계열의 일반 문구로 통일했다.
- 예약 캘린더는 정답 시간뿐 아니라 다른 예약 시간을 확정해도 같은 일반 확인 대화상자가 뜨도록 변경했다.
- 댓글 목록은 목표 댓글뿐 아니라 다른 댓글의 답글 보기나 도움이 돼요 동작에서도 같은 일반 확인 대화상자가 뜨도록 변경했다.
- 검색 결과 목록은 목표 자료뿐 아니라 다른 자료의 미리보기, 저장, 바로 열기에서도 같은 일반 확인 대화상자가 뜨도록 변경했다.
- 따라서 서비스 확인 대화상자가 뜨는 사실만으로 사용자가 정답 여부를 알 수 없도록 했다.

### 2.3 과업 종료 확인 대화상자 추가

- 수행 탭 하단의 `과업 종료` 버튼을 누르면 즉시 종료하지 않고 종료 확인 대화상자를 띄우도록 변경했다.
- 종료 확인 대화상자에는 현재 기록 기준의 최종 수행 결과 문구가 표시된다.
- `예, 종료합니다`를 누르면 현재 상태로 기록을 저장하고 수행 탭을 닫는다.
- `아니요, 계속합니다`를 누르면 종료하지 않고 수행 화면으로 돌아간다.
- 종료 확인 대화상자는 `data-measurement-exempt="true"` 영역으로 처리하여 실제 과업 지표에 포함되지 않게 했다.

### 2.4 최종 수행 결과 문구 분리

종료 확인 대화상자에서만 아래와 같은 판정 문구가 보이도록 했다.

- 성공: `과업 수행에 성공했습니다.`
- 예약 캘린더 오답 예: `요청한 상담 예약 시간과 다른 시간을 예약했습니다.`, `예약 확정 화면에 진입하지 못했습니다.`
- 댓글 목록 오답 예: `요청한 댓글과 다른 댓글의 답글을 열었습니다.`, `요청한 댓글 동작을 완료하지 못했습니다.`
- 검색 결과 목록 오답 예: `요청한 자료와 다른 자료를 저장했습니다.`, `요청한 자료 동작을 완료하지 못했습니다.`

### 2.5 예약 캘린더 B 화면 오류 수정

예약 캘린더 비교안 B에서 예약 시간표를 만들 때 사용하는 `chunkArray()` 함수가 누락되어 있었다. 이 때문에 B 화면 렌더링 중 오류가 발생할 수 있었고, 사용자가 두 번째 화면에 진입했을 때 아무 콘텐츠도 탐색되지 않는 것처럼 보일 수 있었다.

이번 단계에서 `chunkArray()`를 `app.js`에 추가했고, 예약 캘린더 A/B 및 다른 서비스 A/B 수행 화면이 기본 콘텐츠를 렌더링하는지 간이 렌더링 점검을 수행했다.

### 4.4 수행 화면 간이 렌더링 점검

브라우저 자동화 대신 간이 DOM 환경에서 3개 서비스 × 2개 비교안 × 3개 과업, 총 18개 수행 화면 렌더링을 확인했다.

확인 항목:

- 수행 화면 본문이 렌더링되는가
- 오류 화면이 아닌가
- 과업 종료 버튼이 있는가
- 예약 가능 시간 / 댓글 목록 / 검색 결과 같은 주요 콘텐츠가 있는가
- 비교안 B에 최소 하나의 `tabindex="0"` 항목이 있어 키보드 진입점이 있는가
- `live-status-region`이 남아 있지 않은가
- `keyboard-tip` 문구가 남아 있지 않은가

결과: 18개 조합 모두 통과.

## 5. 남은 확인 사항

- 실제 브라우저에서 `과업 종료` → `아니요, 계속합니다` → 계속 수행 흐름을 수동 확인해야 한다.
- 실제 브라우저에서 `과업 종료` → `예, 종료합니다` 뒤 수행 탭이 닫히는지 확인해야 한다.
- 화면낭독프로그램에서 종료 확인 대화상자 초점 가두기와 문구 읽기 순서를 확인해야 한다.
- 이번 단계는 전체 버그 점검 세션이 아니므로, 서비스별 세부 상호작용 오류는 다음 수동 점검에서 이어서 확인한다.
